Configure a COMPANY_NAME server to use SSH authentication, create public and private keys with PuTTYGen, and configure PuTTY to connect to a COMPANY_NAME server

Use SSH Keys with PuTTY

Difficulty: 2
Time: 30 minutes

This article is for Windows users.

Using SSH keys for server logins is more secure and saves time. In this article, you'll do the following:

  • Generate public and private SSH key pairs with PuTTY
  • Load the public SSH keys into the COMPANY_NAME portal
  • Create a new server
  • Create a PuTTY profile
  • Connect to the new server using PuTTY

This article assumes that you have downloaded putty.exe and puttygen.exe.

Generate public and private SSH key pair

  1. Start PuTTYgen.
  2. Click Generate.
  3. Move the mouse within the blank area under Key to generate some randomness until the progress bar is full.
    Once your key is generated, it'll appear in top the field under Public key for pasting into OpenSSH authorized_keys. This is your public key. A properly formatted public key starts with ssh-rss AAAA.
  4. Enter a passphrase in Key passphrase.
  5. Enter the same passphrase in Confirm passphrase.
  6. Click Save private key, and then choose a name and location to save your private key file.
  7. Right-click in the field under Public key for pasting into OpenSSH authorized_keys file, and choose Select All.
  8. Right-click again in the field, and choose Copy.

Load the public SSH key

In this example, we'll get the key from GoDaddy.com.

  1. Go to the SSH key page in the GoDaddy panel - https://cloud.godaddy.com/keys.
  2. Click Create Key. The Create SSH Key window opens.
  3. Enter a unique name that identifies the SSH key you created in the previous step.
  4. Paste the contents (copied in the previous section) in the Key field.
  5. Click OK. A Success message displays.

Create a new server

  1. In the GoDaddy panel, click Servers.
  2. Click Create Server.
  3. Give a unique name to your server, and choose a size and base image.
  4. Enter a username. This username should be different from your main COMPANY_NAME account name.
  5. In the Password field, choose Connect through SSH instead.
  6. From the drop-down list, select the SSH key created in the previous step.
  7. Click Finish.

Create a new PuTTY profile

  1. Start PuTTY.
  2. In the Host Name (or IP Address) field, enter the Public IP Address of the server created in the Godaddy Cloud Servers panel.
  3. In the Port field, enter "22".
  4. On the left-hand side, select the Data subcategory under Connection.
  5. Under Login details, in the Auto-login username field, enter the username created in the previous step.
  6. Under Connection category, expand SSH.
  7. Click Auth.
  8. Make sure that Bypass authetication entirely (the first option) is NOT checked.
  9. On the right-hand side, click Browse under Private key file for authentication.
  10. Browse for, and then select, the private key file created in the previous step.
  11. On the left-hand side, click the Session category (at the top).
  12. Enter a name for the session profile under Saved Sessions, such as My Ubuntu Server.
  13. Click Save.

Connect using your PuTTY profile

  1. Start PuTTY.
  2. Under Saved Sessions, select the profile you created in the previous procedure.
  3. Click Load.
  4. Click Open.
  5. In the PuTTY Security Alert dialog box, click Yes.
  6. When prompted, enter the passphrase used when generating your SSH key.

You are now connected to your server.

Conclusion

In this article, you generated public and private SSH key pairs with PuTTY, loaded your public SSH key into the COMPANY_NAME portal, created a new server, created a new PuTTY profile, and connected to your new server using PuTTY.
