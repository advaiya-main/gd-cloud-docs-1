Describes how to setup a local Django app on Arch Linux

Build a LAMP stack for Python - Debian, Ubuntu

Difficulty: 1
Time: 1.50

Django is a Python web framework that allows rapid development of primarily database-driven web applications. Virtual Environments, or virtualenv, is a Python package that allows development within isolated Python environments. Installing or updating Python packages within one environment has no impact on other environments.

Install virtualenv and virtualenvwrapper

virtualenv is a Python tool used to create isolated environments for Python in which you can install packages without interfering with the other virtualenvs nor with the system Python's packages.

  1. Install virtualenv and virtualenvwrapper.
    sudo pacman -S python2 python2-pip python2-virtualenv python-virtualenvwrapper
  2. virtualenvwrapper requires some initial setup.
    export WORKON_HOME=$HOME/.virtualenvs
    source /usr/bin/virtualenvwrapper.sh

Configure virtual environment

  1. Create the virtualenv
    mkvirtualenv -p /usr/bin/python2.7 myenv
  2. Activate the virtualenv
    workon myenv
  3. Install some package inside the virtualenv
    pip install django pyproj

Install PostgreSQL, GDAL, GEOS, PostGIS

  1. Install PostgreSQL, PostGIS
    sudo pacman -Sy postgresql postgresql-libs postgis
    Select any default values as prompted.
  2. Activate the new virtual environment if not already activated.
    workon myapp
  3. Install Django.
    pip install django

You are ready to create a Django project.

Create a Django project

  1. Create a django project:
    django-admin.py startproject myproject
    This command creates a ./myproject directory containing several projects directories and files.
  2. Navigate into the project directory.
    cd myproject

    The ./myproject/settings.py file can be modified and a preferred database backend configured. Other settings can be modified as necessary. If no database backend is specified, then Django will use the default sqlite3 backend. You can populate with database with some default Django tables.

You can populate with database with some default Django tables.

python manage.py migrate

The project can now be verified by launching the Django web service.

python manage.py runserver

The service should now be accessible athttp://localhost:8000/. Open another terminal while the "runserver" command is running and type this curl command:

curl http://localhost:8000/

The built-in Django web server is NOT meant to be used in a production environment. It is a tool for development only. Once the application is ready to be deployed into production, consider using a web server such as Apache or Nginx with uWSGI.

Step Five - Configure Django App

Once the project is created and operational, it is time to create an application. An application can be created in the same directory as manage.py.

python manage.py startapp myapp

This will create a ./myapp directory with base files ready for your code.

Next steps

Congratulations! You've now got your application deployed and are ready to build dynamic web pages.
