Automate Ruby On Rails Application Deployments using Capistrano - CentOS, Fedora

Difficulty: 2

Time: 30 minutes

In this article, we will assist you to automate the deployment of your Rails application using Capistrano. Capistrano is a remote server automation and deployment tool, which can be used to push your Rails application to remote repository. We are assuming that you have a GitHub account and own sudo priviledges to perform this task.

Prerequisites

If you have not installed ruby with RVM , please visit: Install Ruby on Rails using RVM - CentOS

Capistrano relies on Git for deployments.

Create the script for NGINX Management

  1. Create a new file
    sudo vim /etc/rc.d/init.d/nginx
  2. Add the following lines in this file: 
        #!/bin/sh
	. /etc/rc.d/init.d/functions
	. /etc/sysconfig/network
	[ "$NETWORKING" = "no" ] &amp;&amp; exit 0

	nginx="/opt/nginx/sbin/nginx"
	prog=$(basename $nginx)

	NGINX_CONF_FILE="/opt/nginx/conf/nginx.conf"

	lockfile=/var/lock/subsys/nginx

	start() {
  	  [ -x $nginx ] || exit 5
  	  [ -f $NGINX_CONF_FILE ] || exit 6
  	  echo -n $"Starting $prog: "
 	  daemon $nginx -c $NGINX_CONF_FILE
   	 retval=$?
  	  echo
    	[ $retval -eq 0 ] &amp;&amp; touch $lockfile
    	return $retval
	}

	stop() {
    	echo -n $"Stopping $prog: "
    	killproc $prog -QUIT
   	 retval=$?
   	echo
    	[ $retval -eq 0 ] &amp;&amp; rm -f $lockfile
   	 return $retval
	}

	restart() {
    	configtest || return $?
   	 stop
  	  start
	}

	reload() {
  	  configtest || return $?
   	 echo -n $”Reloading $prog: ”
 	   killproc $nginx -HUP
   	 RETVAL=$?
   	 echo
	}

	force_reload() {
    	restart
	}

	configtest() {
    	$nginx -t -c $NGINX_CONF_FILE
	}

	rh_status() {
   	 status $prog
	}

	rh_status_q() {
    	rh_status >/dev/null 2>&amp;1
	}

	case "$1" in
	start)
	rh_status_q && exit 0
	$1
	;;
	stop)
	rh_status_q || exit 0
	$1
	;;
	restart|configtest)
	$1
	;;
	reload)
	rh_status_q || exit 7
	$1
	;;
	force-reload)
	force_reload
	;;
	status)
	rh_status
	;;
	condrestart|try-restart)
	rh_status_q || exit 0
	;;
	*)
	echo $"Usage: $0 {start|stop|status|restart|condrestart|try-	restart|reload|force-reload|configtest}"
	exit 2
	esac
  3. Save and exit.
    wq!
  4. Provide the following permission for this file:

Capistrano Installation and Create a System User

  1. Install Capistrano
    gem install capistrano
  2. Create a system user for deployment
    sudo adduser UserName
    sudo passwd UserName
  3. Open the /etc/sudoers file
    sudo vim /etc/sudoers

  4. Scroll down to the bottom of this file and find where root is defined. Add the following line.

      UserName ALL=(ALL) ALL
  5. Save and exit.
    wq!

Prepare Rails applications for Git-based Capistrano deployment

Create a basic Ruby-On-Rails Application

  1. Create a new directory and provide all permissions for this directory and move to the newly created directory.
    sudo mkdir –p /var/www
    sudo chmod 777 -R /var/www/
    cd /var/www
  2. Create Rails application in this directory
    rails new HelloWorld
  3. Enter in the application directory
    cd HelloWorld
  4. Migrate the database and generate scaffold task to handle the database.
    RAILS_ENV=development rake db:migrate
    rails generate scaffold Task title:string note:text
  5. Test your application by running the following command in the application directory:
    rails s
  6. For binding to web browser, run the following command:
    rails server --binding=[your droplet's IP]
  7. You should now be able to access it on any browser by typing:
    visiting: http://[your droplet's IP]:3000

Create a Git Repository

  1. Initiate git
    git init
  2. Initiate the repository.
    git add .
    git config --global user.name "GitHub_UserName"
    git config --global user.email "GitHub_Email_Address"
  3. Commit the changes.
    git commit -m "first commit"
  4. Add your GitHub repository link.
    git remote add origin git@github.com: GitHub_UserName/HelloWorld.git
  5. Create the RSA/SSH key.
    ssh-keygen -t rsa
    sudo cat /root/.ssh/id_rsa.pub

    Or

    sudo cat /home/UserName/.ssh/id_rsa.pub
    (for sudo user)
  6. Push the project's source code to your Github account
    git push -u origin master

Work with the Capistrano to automate deployments
