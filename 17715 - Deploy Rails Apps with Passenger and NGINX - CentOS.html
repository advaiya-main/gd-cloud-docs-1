Setting up Rails Apps with Passenger and NGINX in CentOS, Ruby, Rails, NGINX

Deploy Rails Apps with Passenger and NGINX – CentOS

Difficulty: 2

Time: 30 minutes

Rails enables web developer to build rapid application development, while Nginx is a light weight and high performance web server. In this article you will learn to deploy Rails application on Nginx web server using the simple and powerful features of Phusion Passenger. To perform this task, we are assuming that the user has sudo privileges.

Step 1: Install Ruby on Rails using RVM in CentOS

Step 2: Install Nginx and Passenger

  1. Type the following command to install Passenger
    gem install passenger
  2. Before you install Nginx, there are two dependencies that you should run first by typing
    sudo yum install libcurl-devel
    sudo chmod o+x "/home/demo"
    rvmsudo passenger-install-nginx-module
  3. Once you are done with installing Nginx, choose one of the language (in this case, choose Ruby) by hitting the Enter key
    Use <space> to select. If the menu doesn't display correctly, ensure that your terminal supports UTF-8.
    • Ruby
    • Python
    • Node.js
    • Meteor
  4. Next, select number 1 and hit Enter to continue. This action will download, compile and install the Nginx files with Passenger support
    1. Yes: download, compile and install Nginx for me.(recommended)
    The easiest way to get started. A stock Nginx 1.4.4 with Passenger support, but with no other additional third party modules, will be installed for you to a directory of your choice.
    5. Save and exit.

Step 3: Setup a test Rails Application

  1. Create a directory
    sudo mkdir –p /var/www
  2. Move to www directory and create a Rails application
    cd /var/www
    rails new HelloWorld
    cd HelloWorld
  3. Create sample resources
    rails generate scaffold Task title:string note:text
  4. To test that your application, run the following command in the application directory
    ruby -s

Step 4: Configure Nginx

  1. Open the following configuration file nginx.conf
    sudo vim /etc/nginx/conf/nginx.conf
  2. Uncomment the below mentioned line right way
    passenger_root /usr/lib/ruby/vendor_ruby/phusion_passenger/locations.ini;
    passenger_ruby /home/deploy/.rbenv/shims/ruby; # If you use rbenv
  3. Then add the following lines at the end of the nginx.conf file
    server {
    listen 80 default_server;
    listen [::]:80 default_server ipv6only=on;
    server_name mydomain.com; //your server name
    passenger_enabled on;
    rails_env production;
    root/home/deploy/myapp/current/public; //Rails application path

    # redirect server error pages to the static page /50x.html
    error_page 500 502 503 504 /50x.html;
    location = /50x.html
    {
    root html;
    }
    }

    4. Save and exit.

  4. Restart the Nginx service
    sudo service nginx restart
