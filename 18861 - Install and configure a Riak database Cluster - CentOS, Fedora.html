Install and configure a Riak database Cluster - CentOS, Fedora

Difficulty: 2
Time: 1 hour

In this article, you will learn to install a Riak database cluster on CentOS and Fedora servers by compiling from the source. Riak is a distributed database system that is designed for high availability of data. To ensure better performnace, Ubuntu servers with 4 GB RAM is preferred. In this article, you will perform all the installations as a sudo user.

Install dependencies

  1. Install Erlang on all the nodes that you want to include in a cluster.
    yum install gcc gcc-c++ glibc-devel make ncurses-devel openssl-devel autoconf java-1.8.0-openjdk-devel git pam-devel
    yum install wxBase.x86_64
  2. Download Erlang from the source and then compile it.
    wget http://s3.amazonaws.com/downloads.basho.com/erlang/otp_src_R16B02-basho8.tar.gz
    tar zxvf otp_src_R16B02-basho8.tar.gz
    cd OTP_R16B02_basho8
    ./otp_build autoconf
    CFLAGS="-DOPENSSL_NO_EC=1" ./configure && make && sudo make install
  3. Start Erlang using the following command:
    erl

Install Riak

Perform the following installation steps on all nodes that are included in this Riak cluster.

  1. Download Riak from the source and then compile it.
    wget http://s3.amazonaws.com/downloads.basho.com/riak/2.1/2.1.1/riak-2.1.1.tar.gz
    tar zxvf riak-2.1.1.tar.gz
    cd riak-2.1.1
    make locked-deps
    make rel

    Since you have compiled Riak from the source, you need to understand the locations before proceeding further. Below are the file locations:

    • Riak Configuration Directory: Source_File/rel/riak/etc/
    • Binary files : Source_File/rel/riak/bin

Configure the first node

Configure the first node in a Riak cluster by following the below mentioned steps.

  1. Stop the Riak service, if it was started earlier.
    rel/riak/bin/riak stop

    Or, navigate to the Source_File/rel/riak/ director and stop by using the following command:

    bin/riak stop
  2. Edit the Riak configuration file. To do this, find and replace the below lines in Source_File/rel/riak/etc/riak.conf file.
    listener.protobuf.internal = node_ip_address:8087
    listener.http.internal = node_ ip_address:8098
    nodename = riak@node_ip_address
  3. Start the Riak service. While starting, configure the ulimit to minimum of 65565.
    ulimit -u 65565
    bin/riak start

    In the same way, you can configure all other nodes that you want to include in the cluster.

Add a second node to the cluster

Perform all the steps mentioned above to start Riak in second nodes and to include these nodes in the cluster.

  1. Inside the second node, execute the below command to join to the first node:
    rel/riak/bin/riak-admin cluster join riak@first_node_ip
  2. Execute the below steps to plan and commit the changes:
    rel/riak/bin/riak-admin cluster plan
    rel/riak/bin/riak-admin cluster commit
  3. View the complete ring-cluster by executing the below command:
    rel/riak/bin/riak-admin status | grep ring_members

Conclusion

In this article, you have learnt to install multiple nodes, configured them to start Riak and then joined them to the first node to build a Riak database cluster.
