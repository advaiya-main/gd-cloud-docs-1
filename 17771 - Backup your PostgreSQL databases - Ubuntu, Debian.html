Back Up Your PostgreSQL Databases - Debian, Ubuntu

Difficulty: 1

Time: 15 mins

In this article, we will make a backup of a PostgreSQL database that can be used to scale a Ruby on Rails (RoR) application. These instructions assume that you have sudo privileges.

Before starting, make sure that you visit Ruby on Rails is installed and that PostgreSQL is installed.

Back up a PostgreSQL database

You can back up a PostgreSQL database using any of the following methods:

  • Log in with the postgres user, and create a database backup file.
    sudo su - postgres
    pg_dump database_name > backup_file

    Pg_dump is the utility used to dump database information into a file for backup purposes.

  • If you want to perform the backup on a remote system, you can pass the -h flag to specify the remote host, and the -p flag to specify the remote port.
    pg_dump -h remote_host -p remote_port database_name > backup_file
  • If you want to specify a different user, then -U is necessary.
    pg_dump -U user_name -h remote_host -p remote_port database_name >backup_file

Restore data dumps

  1. Create an empty database to store the backup_file.
    createdb -T template0 restored_database_name
  2. Restore a backup created by pg_dump.
    Psql restored_database_name < backup_file
  3. If the test_user owns a table in the database, then create a test_user before restorating the system.
    createuser test_user
    Psql restored_database_name < backup_file

Handle restoration errors

  1. Stop any backup error in PostgreSQL. Using ON_ERROR_STOP command will continue the process without any interruption even if it encounters any error.
    psql --set ON_ERROR_STOP=on restored_database_name < backup_file.
  2. Restore all the database in a single transaction. Using Psql -1 command will stop the process if it encounters any error.
    psql -1 restored_database_name < backup_file

Backup and restore all databases in PostgreSQL

  1. To save time, use the pg_dumpall utility to create a backup for all the databases in the system.
    pg_dumpall > backup_file
  2. Restore the databases with the default database.
    psql -f backup_file postgres
