Setting up Rails application with Unicorn and NGINX in CentOS, Ruby, Rails, NGINX

Deploy Rails App with Unicorn and NGINX – CentOS

Difficulty: 2

Time: 30 minutes

In this article, we will be configuring a multi-layer deployment installation to host Rails based Ruby web applications. To do this, we will use the powerful, flexible, and extremely successful Unicorn application server running behind Nginx.

Assuming that we are logged in as a sudo user.

Prerequisites

In this article, we assume that you have already installed Ruby on Rails with rbenv and your Rails application uses PostgreSQL. If not, please visit:
  • 17521 - Install Ruby on Rails with rbenv – CentOS
  • 17526 - Use PostgreSQL with your Ruby on Rails application – CentOS

Prepare the Deployment Server

  1. Install bundle of development tool by using the following command
    sudo yum groupinstall -y 'development tools'
  2. Install EPEL repository and update the operating system
    sudo wget http://dl.fedoraproject.org/pub/epel/7/x86_64/e/epel- release-7-5.noarch.rpm
    sudo rpm -ivh epel-release-7-5.noarch.rpm
    sudo yum –y update
  3. Install curl-devel and some other dependencies by using the following command
    sudo yum install -y curl-devel nano sqlite-devel libyaml-devel
  4. Install NGINX
    sudo yum install nginx
  5. Install Unicorn
    gem install unicorn

Prepare Rail Application for Deployment

  1. Create a directory and provide all permission for this directory and go to that directory
    sudo mkdir –p /var/www
    sudo chmod 777 -R /var/www/
    cd /var/www/
  2. Create a Rail Application in this directory and move to the application directory
    rails new HelloWorld
    cd HelloWorld/
  3. Create a sample databases and generate a scaffold controller to manage the production database.
    RAILS_ENV=development rake db:migrate
    RAILS_ENV=production rake db:migrate
    rails generate scaffold Task title:string note:text/
  4. Create a directory to hold the PID files
    mkdir pids
  5. Enter the application directory and run the following command
    rails s

    To terminate user, hit CTRL+C.

  6. For binding to web browser, run the following command
    rails server --binding=[your droplet's IP]
  7. We can access it on any browser by visiting
    http://[your droplet's IP]:3000/tasks
  8. Upload source code using following article:

    17414 - Use SFTP to Securely Transfer Files with a Remote Server - ALL DISTRO

Step 3: Configure Server

  1. To configure Unicorn, move to the Rail Application directory
    Cd /var/www/HelloWorld
  2. Create a blank unicorn.rb.
    sudo vim config/unicorn.rb
  3. Add the following code in this file 
    # Set the working application directory
# working_directory "/path/to/your/app"
working_directory "/var/www/HelloWorld"

# Unicorn PID file location
# pid "/path/to/pids/unicorn.pid"
pid "/var/www/HelloWorld /pids/unicorn.pid"

# Path to logs
# stderr_path "/path/to/log/unicorn.log"
# stdout_path "/path/to/log/unicorn.log"
stderr_path "/var/www/HelloWorld/log/unicorn.log"
stdout_path "/var/www/HelloWorld/log/unicorn.log"
	
# Unicorn socket
listen "/tmp/unicorn.[app name].sock"
listen "/tmp/unicorn.myapp.sock"

# Number of processes
# worker_processes 4
worker_processes 2

# Time-out
timeout 30
  4. Save and exit.
    wq!

    You can check the application with Unicorn by running the unicorn_rails inside the app directory.

  5. Configure NGINX and enable it to communicate with Unicorn. To do this, we need to edit the default configuration file: default.conf and leave nginx.conf
    sudo vim /etc/nginx/nginx.conf.default
  6. Replace the file's content with the following lines 
    upstream app {
# Path to Unicorn SOCK file, as defined previously
   	 server unix:/tmp/unicorn.HelloWorld.sock fail_timeout=0;
	}

	server {

   	 listen 80;
   	 server_name localhost;

   	 # Application root, as defined previously
  	  root /root/HelloWorld/public;

  	 try_files $uri/index.html $uri @app;

   	 location @app {
          proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
          proxy_set_header Host $http_host;
          proxy_redirect off;
          proxy_pass http://app;
   	 }

    	error_page 500 502 503 504 /500.html;
     	client_max_body_size 4G;
    	keepalive_timeout 10;
	}
  7. Save and exit.
    wq!

Manage Servers

  1. Once we finish configuring both the servers, initiate the Unicorn and run it as a daemon by using the configuration file. Ensure that we are inside the application directory and run the following command
    unicorn_rails -c config/unicorn.rb –D
  2. Restart the NGINX service
    sudo service nginx restart
  3. Test droplet's IP address (or the domain name associated to it)
    http://Your droplet's IP address/tasks

Conclusion

By using all the above steps we will be able to deploy the Rails application with Unicorn and Nginx on Centos.
