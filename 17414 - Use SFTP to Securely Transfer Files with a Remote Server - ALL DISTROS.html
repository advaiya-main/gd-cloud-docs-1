How To Use SFTP to Securely Transfer Files with a Remote Server

Difficulty: 1
Time: 1.50

How to Connect with SFTP

SFTP uses the SSH protocol to authenticate and establish a secure connection. Because of this, the same authentication methods are available that are present in SSH. Although passwords are easy to use and set up by default, we recommend you create SSH keys and transfer your public key to any system that you need to access. This is much more secure and can save you time in the long run with most or Linux disto.

  1. If you can connect to the machine using SSH, then you have completed all of the necessary requirements necessary to use SFTP to manage files. Test SSH access with the following command:
    ssh username@remote_hostname_or_IP
  2. If that works, exit back out by typing:
    exit
  3. We can establish an SSH connection and then open up an SFTP session using that connection by issuing the following command:
    sftp username@remote_hostname_or_IP
    You will connect the the remote system and your prompt will change to an SFTP prompt.

Getting Help in SFTP

  1. The most useful command to learn first is the help command, You can call it by typing either of these in the prompt:
    help
    ?

Exploring SFTP

We can make our way through the remote system's file hierarchy using a number of commands that function similarly to their shell counterparts.

  1. Type this to find out which directory we are in currently on the remote system
    pwd
    Remote working directory: /home/demouser

    Note: The filenames and home directories are examples used for this document.

  2. We can view the contents of the current directory of the remote system with another familiar command:
    ls
    Summary.txt info.html temp.txt testDirectory
  3. The commands within the SFTP interface are not the normal shell commands and are not as feature-rich, but they do implement some of the more important optional flags:
    ls -la
    drwxr-xr-x    5 demouser   demouser       4096 Aug 13 15:11 .
drwxr-xr-x    3 root     root         4096 Aug 13 15:02 ..
-rw-------    1 demouser   demouser          5 Aug 13 15:04 .bash_history
-rw-r--r--    1 demouser   demouser        220 Aug 13 15:02 .bash_logout

drwx------    2 demouser   demouser       4096 Aug 13 15:04 .cache
-rw-r--r--    1 demouser   demouser        675 Aug 13 15:02 .profile
. . .
  4. To get to another directory issue this command:
    cd testDirectory
