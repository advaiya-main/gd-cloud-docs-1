Setting up local hosts to test virtual host setup on Arch Linux

Set up Local Hosts to Test Virtual Host Setup - Arch Linux

Difficulty: 2
Time: 10 minutes

This article will guide you to test your virtual host setup using local hosts file. You need to first install a webserver (Apache) on your server using Configure Apache Virtual Hosts Setup on Arch Linux link.

Virtual hosts

  • After the Apache is installed and configured on your server, look for the configuration file in /etc/httpd/conf/extra/httpd-vhosts.conf location:
    <VirtualHost *:80>
    ServerAdmin admin@coolexample.com
    ServerName coolexample.com
    ServerAlias www.coolexample.com
    DocumentRoot /var/www/coolexample.com/public_html
    ErrorLog ${APACHE_LOG_DIR}/error.log
    CustomLog ${APACHE_LOG_DIR}/access.log combined
    </VirtualHost>

Set up local hosts

If you want to test your domain before completing the DNS propagation, then with the help of your computer’s hosts file, you can access your configured domain.

  1. If you are using a Linux computer, then edit the hosts file using the following command:
    /etc/hosts
  2. Add appropriate entries for your server’s IP address followed by the domain name, as shown in the following:
    127.0.0.1 localhost
    111.1.11.3 coolexample.com

    The above code tells your local resolver to directly search for the domain in server with IP address - 111.1.11.3. Hence, you will get the contents without complete DNS propagation.

  3. Access the domain http://coolexample.com in your browser to display the page for you.

Conclusion

In this article, you learned to edit your machine’s local host file to resolve your domain.
