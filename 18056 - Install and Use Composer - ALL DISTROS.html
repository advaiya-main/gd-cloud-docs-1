Install and use Composer

Install and Use Composer - ALL DISTROS

Difficulty: 2
Time: 10 minutes

Composer is a tool used to manage program dependencies. This article assumes you have already installed php, git and curl

Install Composer

  1. Make a directory for whatever project composer will be managing
    sudo mkdir /project1
  2. Navigate to this directory
    cd /project1
  3. Run the Composer install script.
    curl -sS https://getcomposer.org/installer | php

    This will create a file, composer.phar in your project directory.

  4. Change name of the newly-created composer.phar file to "composer" (optional)
    sudo mv composer.phar /usr/local/bin/composer
  5. Edit /etc/profile to change your path so that composer will execute; 
    PATH=$PATH:/usr/local/bin/composer
export PATH

Using Composer

There are two ways you can use composer; create a file defining your project or from the command line.

Creating a composer.json file to manage your project.

  1. sudo vim composer.json/
  2. Add the following contents to your file: 
    {
    "require": {
        "vendorname/projectname": "verison [wildcards are supported]"
    }
}
  3. Using laravel as an example: 
    {
    "require": {
        "laravel/laravel": "4.2.*"
    }
}
  4. Now run composer to install dependencies for your laravel project: 
    php composer install

Using create-project

Since the ultimate goal when using Composer is to create a project, it may make sense to simply create the project using Composer's create-project command rather than configure dependencies.

  1. composer create-project [vendor]/[project] {target directory} [version]
  2. Using laravel as our example
    composer create-project laravel/laravel /var/www/laravel 4.2
