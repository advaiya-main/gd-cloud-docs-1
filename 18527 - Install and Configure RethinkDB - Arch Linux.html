Configuring RethinkDB on Arch Linux, NoSQL

Install and Configure RethinkDB - Arch Linux

Difficulty: 2

Time: 15 minutes

RethinkDB belongs to NoSQL database. It's the first distributed document-oriented database. It's designed specifically to push data to applications in real time. This provides a base for creating real-time, scalable, database-oriented web applications. RethinkDB has a very simple API to interact with the databases.

This article discusses installing and configuring RethinkDB on an Arch Linux server named rethinkdb. You should set up a sudo user for each RethinkDB server.

Install RethinkDB software

  1. Update the repository.
    sudo pacman -Sy
  2. Install RethinkDB.
    sudopacman -S rethinkdb

Install the Python client driver

In this article you'll use Python as the desired programming language. Since you have already installed RethinkDB on this server, now you need to install the client driver for connecting to the database from Python.

Python will already be installed on your server. You can use pip, Python Package Manager, to install the client driver for RethinkDB. It's a best practice to run Python applications in an isolated environment.

  1. Install the virtual environment.
    sudo pacman -S python2-virtualenv
    sudo pacman -S python2-pip
    mkdir rethink
    cd rethink
  2. Create the virtual environment structure.
    virtualenv2 venv
  3. Activate the environment.
    source venv/bin/activate

    You can use deactivate to deactivate the current environment.

  4. Install the RethinkDB driver using pip.
    pip install rethinkdb

    This will install the Python client drivers for RethinkDB.

Start RethinkDB

  1. You need to start RethinkDB to access it through the web interface.
    rethinkdb --bind all
  2. You can start by using the above command, or you can make the necessary configuration changes and then start the service as shown below:
    sudo cp /etc/rethinkdb/default.conf.sample /etc/rethinkdb/instances.d/instance.conf
    sudo vi /etc/rethinkdb/instances.d/instance.conf

    Search for:

    server-name= server

    Change it to:

    server-name=rethinkdb
  3. You need to set up the bind address. By default it's accessible only on localhost (127.0.0.1). You need to change this so that it listens on all network interfaces.

    Search for:

    bind=127.0.0.1

    Change it to:

    bind=all

    This will make it listen on all network interfaces.

  4. Save the configuration changes, and then start RethinkDB with a new build configuration.
    sudo systemctl enable rethinkdb@instance
    sudo systemctl start rethinkdb@instance
    sudo netstat -tulnp | grep rethinkdb
    tcp6 0 0 :::28016 :::* LISTEN 2181/rethinkdb tcp6 0 0 :::8080 :::* LISTEN 2181/rethinkdb tcp6 0 0 :::29015 :::* LISTEN 2181/rethinkdb
    Once you complete the above steps, you can access the RethinkDB web interface as your_server_ip_address:8080

    Using this Web interface, you can manage the creation of databases and tables, and you can configure the RethinkDB cluster to manage table replication.

Interact with RethinkDB from Python

In most cases, you'll be managing databases with the programming language. The following commands will help you.

  1. Activate the virtual environment for Python.
    cd ~/rethink
    source venv/bin/activate
  2. Start Python interpreter.
    python
  3. Import the client driver into the environment.
    import rethinkdb as r
  4. Connect with the local database.
    r.connect("localhost",28015).repl()
  5. Create a database.
    r.db_create("article").run()
  6. Create table posts.
    r.db("article").table_create("posts").run()
  7. Insert values into table posts.
    r.db("article").table("posts").insert({"hello":"world"}).run()
  8. Print the contents in a table.
    c = r.db("food").table("favorites") for x in c: print x
  9. You can use .filter() to filter out from the retrieve query.
