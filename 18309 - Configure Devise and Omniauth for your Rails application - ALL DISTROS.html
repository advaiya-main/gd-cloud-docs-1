Configure Devise and OmniAuth for your Rails Application - ALL DISTROS

Difficulty: 3
Time: Less than 20 minutes

Devise and OmniAuth are the two most popularly used user registration and authentication mechanism in most Ruby on Rail applications. Developing these mechanisms right from the beginning needs a lot of effort and time. In such situation, you can use Devise gem. Using this you can set up an authentication mechanism within minutes. If you allow your users to login with their existing Facebook, Amazon or Twitter accounts, you can expect them to be much happier. You can perform this with the help of OmniAuth gem.

In this tutorial, you will install Devise and OmniAuth gem with your Ruby application. You should setup a sudo user for each RethinkDB server, as working with a sudo user is a good security practice.

Before starting with the installation steps, you should have a server (with any Linux version) with Ruby installed through RVM (Ruby Version Manager). You can refer the following links to know more about installing Ruby using RVM:

  • Install Ruby on Rails using RVM – Ubuntu
  • Install Ruby on Rails using RVM - Fedora
  • Install Ruby on Rails using RVM - CentOS
  • Install Ruby on Rails using RVM - Arch Linux
  • Install Ruby on Rails using RVM - Debian

Create a Rails application

  1. Install Rails application on a separate directory, before adding a new Rails application. And this is always recommended.
    mkdir rails_applications
    cd rails_applications
  2. Start the RVM script from ~/.rvm/scripts/rvm. Since you are using sudo users for the purpose installation, RVM will be installed in user’s home directory.
    Source ~/.rvm/scripts/rvm
    rvm use ruby –default
  3. Create a new application. You may name it as myapp.
    rails new myapp
    cd myapp
  4. Run all Rails installations inside your application directory ~/.rails_applications/myapp.

Add the required Gems to the Gemfile

In this section, you will need Devise and OmniAuth gems. Along with this, you also need a separate gems for every OAuth service provider which you would like to add. In this tutorial, you will login through Facebook for which omniauth-facebook gem is required for Facebook authentication.

  1. Add the following lines to the end of the file ~/rails_apps/myapp/Gemfile:
    gem 'therubyracer'
    gem 'devise'
    gem 'omniauth'
    gem 'omniauth-facebook'
  2. Refer few of the popular authentication gems as listed:
    • Twitter : omniauth-twitter
    • Amazon : omniauth-amazon
    • Google : omniauth-google
    • Github : omniauth-github
  3. Install the newly added gems.
    bundle install

Add the application for testing

Here, you will add few pages for testing. These pages will be available after you login into your application.

  1. Use Rails' scaffolding feature to add these pages. With rails g scaffold command, you just need to specify the model and Ruby will generate the fully functional pages for you.
    rails g scaffold Product name:string price:integer description:text
    rake db:migrate
  2. Define the root of this application. To add the root of the application, you need to edit the file ~/.rails_applications/myapp/config/routers.rb and then append the line root 'products#index'. 
    Rails.application.routes.draw do
  resources :products
  root 'products#index'
end
  3. Start your application sever to test the features which you have added just now. This will start the application in your localhost.
    rails s
  4. Access this application in either of the ways - using localhost (http://localhost:3000) or through the IP address on which the application is binded to(http://ip_address:3000).
    rails s –bindings=ip_address
  5. Add new products and this will be saved in your application.
  6. Run Ctrl + C to stop the development server.

Authenticate through Devise

  1. Run the following commands to add Devise authentication support. This will create the sign-in and sign-up forms.
    rails generate devise:install
    rails generate devise User
    rake db:migrate
  2. Edit ~/rails_applications/myapp/app/controllers/application_controller.rb and add authenticate_user!, so that all pages in the application will be accessible only after the application login. Below is the code: 
    class ApplicationController < ActionController::Base
  protect_from_forgery with: :exception
  before_action :<authenticate_user!>
end
  3. Start the development server and access it in a browser.
    rails s
    http://localhost:3000
  4. Enter the particular IP address if required:
    rails s –bindings=ip_address
    http://ip_address:3000
  5. Log in or sign up using the signup link on this login page.

Add OmniAuth support

  1. Close the application session by Ctrl-C and then add OmniAuth. Here you will add another column named uid and provider string to the model that was generated by Devise.
    rails g migration AddColumnsToUsers provider uid
    rake db:migrate
  2. Contact OAuth provider to get the Client ID and Secret ID , and submit their forms accordingly. You will be asked for a callback URL, which is separate for each provider. Here are a few of them for your reference:
    • Facebook: http://localhost:3000/users/auth/facebook/callback
    • Amazon: http://localhost:3000/users/auth/amazon/callback
    • Twitter: http://localhost:3000/users/auth/twitter/callback
    • Google: http://localhost:3000/users/auth/google/callback
  3. Declare the provider in ~/rails_applications/myapp/config/initializers/devise.rb.
  4. Add the Client ID and the Secret ID. 
    #Add your ID and secret here
  #ID first, secret second
  config.omniauth :<facebook, "Client ID", "Secret ID">
end
  5. To set an email address, you can add the same in the configuration file using:
    config.mailer_sender = 'mailer@example.com'

Update model

  1. Make your model omniauthable. For that you need to change the service providers in the user model. To do this, edit file ~/rails_applications/myapp/app/models/user.rb and add the below lines: 
    class User < ActiveRecord::Base
  devise :database_authenticatable, :registerable,
         :recoverable, :rememberable, :trackable, :validatable,
     :omniauthable, :omniauth_providers => [:facebook]
def self.from_omniauth(auth)
      where(provider: auth.provider, uid: auth.uid).first_or_create do |user|
        user.provider = auth.provider
        user.uid = auth.uid
        user.email = auth.info.email
        user.password = Devise.friendly_token[0,20]
      end
  end
end
  2. Save and exit the file by pressing the Esc key, and then typing :wq, followed by the Enter key.

Handle callback URLs

  1. Specify the controller that will be handling your call back requests. To do this, edit the below file and update devise_for line.
    sudo vim ~/rails_applications/myapp/config/routes.rb
  2. Add the line :controllers => { :omniauth_callbacks => "callbacks" } next to devise_for : users. 
    Rails.application.routes.draw do
  devise_for :users, :controllers => { :omniauth_callbacks => "callbacks" }
  resources :products
  root 'products#index'
end
  3. Create a new file at ~/rails_applications/myapp/app/controllers/callbacks_controller.rb.
  4. Add the following code in the newly created file: 
    class CallbacksController < Devise::OmniauthCallbacksController
    def facebook
        @user = User.from_omniauth(request.env["omniauth.auth"])
        sign_in_and_redirect @user
    end
end

If you want more OAuth providers, then you must have separate methods for them. It is mandatory that the name of the method and the provider name are same. For example, for adding Twitter support, the method will be defined using def twitter. Since you have made all the required configurations, you can now start the development server.

rails s

Login into your application with your Facebook account now.

Conclusion

In this article, you reviewed the steps involved in configuring Devise and OmniAuth on all distros. This way you can login into your applications with your Facebook login details.
