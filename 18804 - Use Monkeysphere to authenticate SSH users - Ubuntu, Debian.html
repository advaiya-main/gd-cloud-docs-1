Use Monkeysphere to authenticate SSH users - Debian, Ubuntu

Difficulty: 3
Time: 2.5 hours

Introduction

In this article you will authenticate ssh users through Monkeysphere that will allow you to have a trusted SSH login.

Before starting this section, you need to have Monkeysphere installed and configured

Create an SSH Server identity certifier

Identity certifier is an entity that is trusted, and ensures that users are trusted. In this article, admin server is used as the identity certifier.

  1. Get the fingerprint of the administrative user.
    gpg --with-colons --fingerprint admin@junkdomain.com
  2. Locate and note the short fingerprint, which is a long alphanumeric string next to fpr in the output from above command. In this case, it is 99981C07DC4177F646AFF531DEA69AE3462908AF. 
    tru::1:1441542079:0:3:1:5
pub:u:2048:1:DEA69AE3462908AF:2015-09-05:::u:admin <admin@junkdomain.com>::scESC:
fpr:::::::::99981C07DC4177F646AFF531DEA69AE3462908AF:
sub:u:2048:1:D5542FF062F4D3E5:2015-09-05::::::e:
  3. Add the fingerprint in the SSH server. With this, Monkeysphere will get the GPG key from key server that will be added to its key ring.
    monkeysphere-authentication add-identity-certifier 99981C07DC4177F646AFF531DEA69AE3462908AF

Generate the SSH user authentication subkeys

All the client machines need to generate a GPG sub key. This key will be used for user authentication of the server. In case you already have a GPG key, that will be used to identify and authenticate the user.

  1. Generate sub keys on client servers using the Monkeysphere command.
    monkeysphere gen-subkey

    Once the sub key is generated, it is added to the local GPG key ring of client server. This key generation process will take few minutes.

  2. List out the sub key in client key ring.
    gpg --list-keys client@junkdomain.com
  3. Note down the short key from the output, as shown in the highlighted content below. For this article, it is 87791BD0. 
    pub   2048R/87791BD0 2014-03-14
uid                  client <client@junkdomain.com>
sub   2048R/3294D31D 2014-03-14
sub   2048R/0FECF512 2014-03-14
  4. Upload the key change to the key server.
    gpg --keyserver pool.sks-keyservers.net --send-key 87791BD0

Create authentication files on SSH server

Monkeysphere uses some authentication files to generate actual authentication files for SSH server. To create the authentication file, perform the following steps on the SSH Server:

  1. Create the monkeysphere directory.
    mkdir .monkeysphere
  2. Grant required permissions.
    chmod 755 .monkeysphere
  3. Move to the monkeysphere directory.
    cd .monkeysphere
  4. Create a new file named authorized_user_ids and open it for editing.
    vim authorized_user_ids
  5. On the client server, get the information about the client user (name and email).
    gpg --list-keys client@junkdomain.com
  6. Locate and note the client user name from the output. This information is exactly in the same format as required in the file authorized_user_ids above. 
    pub   2048R/87791BD0 2015-08-20
uid              client client@junkdomain.com
sub   2048R/3294D31D 2015-08-20
sub   2048R/0FECF512 2015-08-20
  7. On the admin server, get the information about the admin user (name and email).
    gpg --list-keys client@junkdomain.com
  8. Locate and note the admin user name from the output. This information is exactly in the same format as required in the authorized_user_ids above. 
    pub   2048R/87791BD0 2015-08-20
uid              admin admin@junkdomain.com
sub   2048R/3294D31D 2015-08-20
sub   2048R/0FECF512 2015-08-20
  9. Add the information collected from above the two steps and put them in the same format in the file authorized_user_ids on the SSL server server.
    client <client@junkdomain.com>
    admin <admin@junkdomain.com>
  10. Save and exit the file by pressing the Esc key, and then typing :wq, followed by the Enter key.
  11. Provide proper permission for the file.
    chmod 644 authorized_user_ids

Create internal authentication files

  1. Create internal authentication files by using the following command:
    monkeysphere-authentication update-users

    In case, any particular user needs to be updated, the following command can be used:

    monkeysphere-authentication update-users user_name
  2. Navigate to /var/lib/monkeysphere/authorized_keys directory.
    cd /var/lib/monkeysphere/authorized_keys
  3. List to the contents of the directory to ensure that generated authentication file exists.
    ls

    A file named user should be available.

  4. Open the file for viewing.
    cat user
  5. Note that the file contains the key, like normal authorized_keys entries.

  6. Specify the authorized key location in ssh configuration file:
    sudo vi /etc/ssh/sshd_config
  7. Locate the AuthorizedKeysFile parameter and update it as follows:
    AuthorizedKeysFile /var/lib/monkeysphere/authorized_keys/%u
  8. Save and exit the file by pressing the Esc key, and then typing :wq, followed by the Enter key.
  9. Restart ssh service.
    service ssh restart

Setup client for authentication through GPG key

  1. On the client server, execute the below command:
    ssh-agent sh -c 'monkeysphere subkey-to-ssh-agent && ssh server.example.com'

    You will be asked to enter the key passphrase and no user password will be prompted. After entering the key pass phrase, you will be logged in to server server.

  2. Set up an agent session.
    eval $(ssh-agent)
  3. Edit the bash_profile file to start the session automatically.
    echo 'eval $(ssh-agent)' >> ~/.bash_profile
  4. Add the GPG subkey to the client.
    monkeysphere subkey-to-ssh-agent
  5. Provide the password upon prompt. This will save the password for the entire session.
  6. Check that the password has been accepted using the following command:
    ssh-add –l

    Successful execution of the process would result into the following output:

    2048 2a:1a:1d:52:32:e5:f4:45:b2:a3:ff:d0:c0:6e:69:f6 client <client@junkdomain.com> (RSA)

Conclusion

In this article, you learned the procedure to authenticate user through Monkeysphere. When using Monkeysphere, clients do not need to worry about the authenticity of the server to which it is connecting to.
