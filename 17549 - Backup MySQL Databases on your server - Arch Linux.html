Backup MySQL Databases on your server - Arch Linux

MySQL is one of the most popular database engine for storing data in database. MySQL uses SQL ie. Structured Query Language to retrieve and manipulate data. It can be used to manage data from websites or applications.

Backup is useful to restore data against crashes of the system. In this article, you will learn ways to backup MySQL databases on your Arch Linux server. You can choose any version of MySQL, as operational commands remain the same in all versions.

Backup the database

  • Export the database using mysqldump command.
    mysqldump -u mysql_username -p mysql_user_password database_name > back_database_date.sql

Restore the database

  1. Create a database if not created already.
    mysql -u mysql_username -p
  2. You will get a MySQL prompt, asking to create a new database to hold raw data.
    CREATE DATABASE database_name;
  3. Exit or press Ctrl+c.
  4. Import data in the newly created database.
    mysql -u mysql_username -p database_name < back_database_date.sql

    The data will be restored or dumped into database_name.

Back up the table to a text file

  1. Take the backup into the text file by using the following command:
    USE database_name;
    SELECT * INTO OUTFILE 'table_backup_file' FROM name_of_table;

    Use USE command to select or change the database.

  2. The default location of the file is /var/lib/mysql/database_name/file_name. Verify the output by using the following command with the sudo user:
    sudo ls /var/lib/mysql/database_name

Auto-backup MySQL information

  1. To automatically perform data backups at regular intervals, there is a tool in Arch Linux named automybackup. Install automybackup using the following command:
    wget http://downloads.sourceforge.net/project/automysqlbackup/AutoMySQLBackup/AutoMySQLBackup%20VER%203.0/automysqlbackup-v3.0_rc6.tar.gz?r=http%3A%2F%2Fsourceforge.net%2Fprojects%2Fautomysqlbackup%2F%3Fsource%3Dtyp_redirect&ts=1440596561&use_mirror=netassist

  2. Rename the downloaded filename if you find the name long.

  3. Extract the file.
    tar zxvf automysqlbackup-v3.0_rc6.tar.gz
  4. Open the configuration file for automysqlback using vim which is located at /etc/default/automysqlbackup.
    sudo vim /etc/automysqlbackup/automysqlbackup.conf
  5. Using the below configuration variables, perform the necessary settings to access MySQL.
    CONFIG_mysql_dump_username = 'mysql_username'
    CONFIG_mysql_dump_password = 'password'
    CONFIG_mysql_dump_host_friendly='localhost' #optional
    CONFIG_backup_dir = '/path/to/backup'
  6. To execute automysqlbackup command, change the directory where you extracted the file. Find the command and execute it using the following command:
    ./automysqlbackup
  7. To verify the database backup, run the following command in the terminal:
    ls -R /path/to/backup

Backup using replication

You may use MySQL replication to backup the data with the above techniques. While replication allows for data mirroring, however it might fail to save a specific point in time. This is because it is constantly replicates the changes in a dynamic environment. To avoid this problem, you can either disable replication temporarily or create the backup machine in a read-only state temporarily

Disable replication temporarily

  1. Disable replication for the slave temporarily by using the following command:
    mysqladmin -u mysql_username -p stop-slave
  2. Another option, that doesn't completely stop replication, but put pauses the replication can be accomplished, by typing:
    mysql -u mysql_username -p -e 'STOP SLAVE SQL_THREAD;'

Making the backup machine read-only temporarily

You can ensure a consistent set of data within the server by making the data read-only temporarily. You can perform these steps on either the master or the slave systems.

  1. Log into MySQL with enough privileges to manipulate the data.
    mysql -u mysql_username -p
  2. You can write all of the cached changes to the disk and set the system read-only by typing:
    FLUSH TABLES WITH READ LOCK;
    SET GLOBAL read_only = ON;
  3. Perform the backup using mysqldump.
  4. After the backup is completed, return the system to its original working state by typing:
    SET GLOBAL read_only = OFF;
    UNLOCK TABLES;
