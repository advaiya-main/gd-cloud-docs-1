Install and Configure Puppet to Manage Your Server - Fedora

Difficulty: 4

Time: 30 minutes

Puppet is a configuration management system that helps to automate configuration and management of server infrastructure.

Puppets are of two types:

  • Puppet Enterprise
  • Open source Puppet

In this article, you will learn to install open source puppet in a Master-Agent server setup. This includes a Puppet Master server which has all the configuration files and Puppet Agent server which will be querying for Master server for updates.

Before you begin, you must have root access to follow this article. In addition, you should have the below mentioned infrastructure ready:

  • Server with Fedora 20 installed that will act as the Master server.
  • A private network DNS.
  • Ensure that the firewall port 8140 is accessible.

Throughout this article, the below naming conventions will be used:

  • For puppet master server hostname will be puppet and corresponding FQDN will be puppet.coolexample.com
  • For host nodes, it is host1 and host2 and its FQDN host1.coolexample.com and host2.coolexample.com

Install Network Time Protocol (NTP)

If time is not properly adjusted, then it can cause errors when puppet master issues agents certificates.

  1. Install the NTP daemon so that time gets automatically updated.
    sudo yum install ntp
    sudo ntpdate pool.ntp.org
  2. Edit ntp.conf file.
    sudo vi /etc/ntp.conf
  3. Add time servers from the NTP Pool Project page.
    server 0.centos.pool.ntp.org iburst server 1.centos.pool.ntp.org iburst server 2.centos.pool.ntp.org iburst server 3.centos.pool.ntp.org iburst
  4. Start the NTP service.
    sudo systemctl enable ntpd
    sudo systemctl start ntpd

    5. Server will have accurate time now.

Install Puppet master

  1. Get the correct version of yum repository.
    sudo rpm -ivh https://yum.puppetlabs.com/puppetlabs-release-fedora-20.noarch.rpm
  2. Install the package.
    sudo yum install puppet-server
  3. Do not start the puppet service now.

Lock Puppet version

If the puppet server is updated or changed to another version, then it will stop working. For gaining consistency, you need to maintain a single version in the server. In a puppet infrastructure, if agents have higher version number, then the master will not be able to manage the same. For this reason, you need to lock the current version.

  1. View the current version.
    puppet help | tail -n 1
  2. Install the package named yum-versionlock to lock the version of puppet.
    sudo yum install yum-versionlock
  3. Lock the puppet version.
    sudo yum versionlock puppet
    sudo yum versionlock puppet-server

Names and certificates

Communication between puppet master and agent is being authenticated through SSL Certificates. Follow the below mentioned steps to generate the master SSL certificate:

  1. Delete existing certificates from /var/lib/puppet/ssl.
    sudo rm -rf /var/lib/puppet/ssl
  2. Edit /etc/puppet/puppet.conf file, in order to configure the certificate.
    sudo vi /etc/puppet/puppet.conf
  3. It will look like:
    [main] logdir = /var/log/puppet rundir = /var/run/puppet ssldir = $vardir/ssl [agent] classfile = $vardir/classes.txt localconfig = $vardir/localconfig
  4. Add the below lines in this file:
    [master] certname = puppet dns_alt_names = puppet,puppet.example.com
  5. Generate a new certificate.
    sudo puppet master --verbose --no-daemonize
  6. This will generate a new certificate for the puppet master. After running this command, you will see this line, Notice: Starting Puppet master version 3.8.2. This means that execution is completed. Press Ctrl + c to end the process.

Configure Puppet master

  1. Edit configuration file at /etc/puppet/puppet.conf for configuring the Puppet.
    sudo vi /etc/puppet/puppet.conf
  2. Puppet has files called manifest, which contains the description about system configuration. Create a default manifest:
    sudo touch /etc/puppet/manifests/site.pp

Install Puppet agent

Configure DNS properly and make sure that puppet master and agent resolve properly. Follow the same procedure in all agents.

  1. Install PuppetLab yum repository.
    sudo rpm -ivh https://yum.puppetlabs.com/puppetlabs-release-fedora-20.noarch.rpm
  2. Install the Puppet agent package.
    sudo yum install puppet

Lock puppet version

  1. View the current version.
    puppet help | tail -n 1
  2. Install the package named yum-versionlock to lock the version of the puppet.
    sudo yum install yum-versionlock
  3. Lock the puppet version.
    sudo yum versionlock puppet
  4. Your puppet version is locked now.

Set up agent

  1. Edit the configuration file.
    sudo vi /etc/puppet/puppet.conf
  2. Add the below statement to the section:
    [agent] server = puppet.example.com
  3. Save and exit the file.
  4. Start the agent.
    systemctl list-unit-files | grep puppet
    sudo puppet resource service puppet ensure=running enable=true
    systemctl list-unit-files | grep puppet puppet.service enabled puppetagent.service disabled

  5. This will configure host1 server with puppet. Like this you can edit all the host servers to join puppet master server.

Sign request on master

As soon as the puppet starts in agent node, it will send a certificate to master server. The master signs this certificate and then it will be able to manage the agent node. Below steps describe the full process:

List certificates

  1. View the unsigned certificate in puppet master by running the following command:
    sudo puppet cert list
  2. If you have one agent node, then you will have one request. It contains the hostname of the agent node.
    "host1.coolexample.com" (SHA256) B1:96:ED:1F:F7:1E:40:53:C1:D4:1B:3C:75:F4:7C:0B:A9:4C:1B:5D:95:2B:79:C0:08:DD:2B:F4:4A:36:EE:E3

Sign a request

  1. Use the puppet cert sign command to sign an agent nodes certificate in master server.
    sudo puppet cert sign host1.coolexample.com
  2. The output is as follow:
    Notice: Signed certificate request for host1.example.com Notice: Removing file Puppet::SSL::CertificateRequest host1.example.com at '/var/lib/puppet/ssl/ca/requests/host1.example.com.pem'

  3. At this point puppet master server can communicate and control agent nodes which are signed through certificates.

  4. Sign the entire request.
    sudo puppet cert sign --all

Revoke certificate

  1. Remove or rebuild a host from puppet. Then you need to revoke the host certificate.
    sudo puppet cert clean hostname
  2. View all signed certificate:
    sudo puppet cert list --all

Get started with Puppet

All above steps are complete and you have your puppet master-agent infrastructure setup and running.

Gather facts

Puppet has a tool called Facter. Using this tool puppet gathers information about all agent nodes running in infrastructure. By default it gathers all system configuration information.

  • View the list of factors.
    facter

Excecute main manifest

Puppet agent is configured to periodically check with puppet master. During this time, puppet agent will send facts about itself to master and then it pulls a list of resources from master, which are determined by its main manifest.

Execute agent node

  1. Run the below command on the agent to check with the master:
    sudo puppet agent -t

    or

    sudo puppet agent --test
  2. It will show the following details:
    Info: Retrieving pluginfacts Info: Retrieving plugin Info: Loading facts in /var/lib/puppet/lib/facter/pe_version.rb Info: Loading facts in /var/lib/puppet/lib/facter/puppet_vardir.rb Info: Loading facts in /var/lib/puppet/lib/facter/root_home.rb Info: Loading facts in /var/lib/puppet/lib/facter/facter_dot_d.rb Info: Caching catalog for host1.example.com Info: Applying configuration version '1407966507'
  3. Using this command, you can see the number of manifest that will affect a server or node immediately.

Use one-off manifests

Using puppet apply command, you can execute a manifests which are not related to the main manifest. This is isolated to the node on which this command is executed.

sudo puppet apply /etc/puppet/modules/test/init.pp

Use a simple manifest

  1. Edit the file at /etc/puppet/manifests/site.pp that contains the puppet master main manifests.
    sudo vi /etc/puppet/manifests/site.pp
  2. Add the following lines:
    file {'/tmp/<em><u>example-ip</u></em>': ensure => present, mode => 0644, content => "Here is my Public IP Address: ${ipaddress_eth0}.\n", }
  3. Save the file and exit.
  4. Execution of the above line will create a file named /tmp/example-ip and this will contain the public IP address of the node.
  5. Either you can wait for the agent node to check this change through routine checkups or you can execute manually as:
    sudo puppet agent -t

    Or

    sudo puppet agent --test

Define a node

  1. You can define a specific node, by editing the file at /etc/puppet/manifests/site.pp.
    sudo vi /etc/puppet/manifests/site.pp
  2. Add the following lines:
    node 'ns1', 'ns2' { file {'/tmp/dns': ensure => present, mode => 0644, content => "Only DNS servers get this file.\n", } } node default {}
  3. Save and exit the file.
  4. Execute the command sudo puppet agent -test or wait for the agent pull from the file /tmp/dns.

Use modules

  1. Modules are used to group tasks together. You can get different modules in puppet community. Here you will install puppetlabs-apache module.
    sudo puppet module install puppetlabs-apache
  2. Edit the file.
    sudo vi /etc/puppet/manifest/site.pp
  3. Add the following lines:
    node 'host2' { class { 'apache': } apache::vhost { 'example.com': port => '80', docroot => '/var/www/html' } }
  4. Save and exit the file.
  5. Above code will make host2 to install the Apache package and will make it listening on port 80. To execute manually, run the following command:
    sudo puppet agent -t

    Or

    sudo puppet agent --test
