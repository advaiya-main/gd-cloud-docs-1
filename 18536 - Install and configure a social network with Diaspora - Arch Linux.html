Install and Configure a Social Network with Diaspora - Arch Linux

Difficulty: 3
Time: 20 minutes

Diaspora is an open-source social network that is used in a distributed network. In this article, you'll learn how to install and configure a social network using Diaspora on an Arch Linux droplet. For security reasons, you should always prefer to use sudo user login.

Before you start, make sure that the droplet has at least 1 GB of RAM and that the domain name points to the server. In addition, you'll need to install an SSL certificate on your web server via SSH on the Arch Linux droplet. If you need to create a swap file, visit Add Memory Swap on Arch Linux.

Install the dependencies

Diaspora needs both ruby2.1 and ruby2.1-bundler versions for installation purpose.

  1. Synchronize to package repository and install the basic packages.
    sudo pacman -Sy
    sudo pacman -S --needed base-devel
  2. Install ruby2.1 version from Arch User Repository (AUR).
    mkdir build
    cd build
    wget https://aur.archlinux.org/cgit/aur.git/snapshot/ruby2.1.tar.gz
    tar -xvzf ruby2.1.tar.gz
    cd ruby2.1
    makepkg -s
    sudo pacman -U ruby2.1-2.1.7-1-x86_64.pkg.tar.xz
  3. Install ruby2.1-bundler from Arch User Repository (AUR).
    cd /home/user/build
    wget https://aur.archlinux.org/cgit/aur.git/snapshot/ruby2.1-bundler.tar.gz
    tar -xvzf ruby2.1-bundler.tar.gz
    cd ruby2.1-bundler
    makepkg -s
    sudo pacman -U ruby2.1-bundler-1.10.6-1-any.pkg.tar.xz

Set up and secure database for Diaspora

Following steps will install MariaDB on Arch linux.

  1. Download the source code from AUR and install the same.
    cd /home/user/build
    wget https://aur.archlinux.org/cgit/aur.git/snapshot/diaspora.tar.gz
    tar -xzvf diaspora.tar.gz
    cd diaspora
    makepkg -s
    sudo pacman -U diaspora-mysql-0.5.3.1-1-x86_64.pkg.tar.xz
  2. Initialize and start the MySQL service.
    sudo mysql_install_db --user=mysql --basedir=/usr --datadir=/var/lib/mysql
    sudo systemctl enable mysqld.service
    sudo systemctl start mysqld.service
  3. Secure MariaDB installation by executing the following command:
    sudo mysql_secure_installation

Create a new user and database for Diaspora

  1. After MariaDB is installed, create a database for diaspora by logging into MariaDB.
    mysql -u root –p
  2. Run the following queries inside the MariaDB:
    CREATE USER 'diaspora'@'localhost' IDENTIFIED BY 'password';
    CREATE DATABASE IF NOT EXISTS `diaspora_production` DEFAULT CHARACTER SET `utf8mb4` COLLATE `utf8mb4_bin`;
    GRANT SELECT, INSERT, UPDATE, DELETE, CREATE, DROP, INDEX, ALTER, LOCK TABLES ON `diaspora_production`.* TO 'diaspora'@'localhost';
    \q
  3. Log in as a database user in Diaspora.
    mysql -u diaspora -p -D diaspora_production
    \q

Set up Diaspora settings

  1. Switch to Diaspora user and configure the basic settings.
    sudo su diaspora
    cd
  2. Edit database.ymlfile and give proper entries with respect to PostgreSQL database.
    vim /etc/webapps/diaspora/database.yml
  3. Make following changes in the config/diaspora.yml file:
    vim /etc/webapps/diaspora/diaspora.yml
    • Uncomment line 39, and add a proper https URL.
    • Uncomment line 47, and choose certificate_authorities.
    • Uncomment line 177, and edit as follows:
      rails_environment: 'production'
  4. Save and close the file by pressing the Esc key and then typing :wq, followed by pressing the Enter key.
  5. Initialize the schema.
    bin/bundle exec rake db:create db:schema:load
  6. Switch back to regular user and start the Diaspora service in droplet.
    su - user
    sudo systemctl enable diaspora.service
    sudo systemctl start diaspora.service

Conclusion

In this article, you have learnt how to install and configure social network using Diaspora on an Arch Linux droplet.
