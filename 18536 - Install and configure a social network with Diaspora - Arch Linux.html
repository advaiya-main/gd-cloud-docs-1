Install and configure a social network with Diaspora - Arch Linux

Difficulty: 3
Time: 20 minutes

Diaspora is an open-source social network which is used in a distributed network and not centralized.

In this article, you will learn how to install and configure social network using Diaspora in an Arch Linux droplet. For security, you should always prefer sudo user login.

Prerequisites

Before you start you should satisfy below requirements:

  • Droplet with minimum 1 GB Ram
  • Sudo users
  • SSL certificate

You need to have SSL certificate purchased for running Diaspora. SSL certificate is mandatory. Self-signed certificates are not recommended. You can refer below URL to know about installing SSL certificate in your webserver.

Install an SSL certificate on your web server via SSH - Arch Linux

  • Domain name pointing to your server
  • Swap file

You can refer this URL 17329-Add Memory Swap – Arch Linux to know about creating a swap file.

Install dependencies

Diaspora needs ruby2.1 and ruby2.1-bundler versions for installation purpose. It is mandatory that Ruby version is 2.1

Execute below steps to install all dependencies.

  1. Synchronize to package repository and install the basic packages.
    sudo pacman -Sy
    sudo pacman -S --needed base-devel
  2. Install Ruby2.1 version from AUR.

    You need to install Ruby2.1 version from AUR. For that execute these steps:

    mkdir build
    cd build
    wget https://aur.archlinux.org/cgit/aur.git/snapshot/ruby2.1.tar.gz
    tar -xvzf ruby2.1.tar.gz
    cd ruby2.1
    makepkg -s
    sudo pacman -U ruby2.1-2.1.7-1-x86_64.pkg.tar.xz
  3. Install Ruby2.1-bundler from AUR.

    You need to install Ruby2.1-bundler version from AUT. To achieve this execute below steps:

    cd /home/user/build
    wget https://aur.archlinux.org/cgit/aur.git/snapshot/ruby2.1-bundler.tar.gz
    tar -xvzf ruby2.1-bundler.tar.gz
    cd ruby2.1-bundler
    makepkg -s
    sudo pacman -U ruby2.1-bundler-1.10.6-1-any.pkg.tar.xz

Install and secure database for Diaspora

Following steps will install MariaDB on Arch linux.

  1. Download the source code from AUR and install the same.
    cd /home/user/build
    wget https://aur.archlinux.org/cgit/aur.git/snapshot/diaspora.tar.gz
    tar -xzvf diaspora.tar.gz
    cd diaspora
    makepkg -s
    sudo pacman -U diaspora-mysql-0.5.3.1-1-x86_64.pkg.tar.xz
  2. Start MySQL service.

    Initialize and start service:

    sudo mysql_install_db --user=mysql --basedir=/usr --datadir=/var/lib/mysql
    sudo systemctl enable mysqld.service
    sudo systemctl start mysqld.service
  3. Secure MariaDB installation by executing the following command:
    sudo mysql_secure_installation

Add Diaspora user in database

Once MariaDB is installed, create database for diaspora.

  1. Log in to MariaDB.
    mysql -u root –p
  2. Run below queries inside MariDB:
    CREATE USER 'diaspora'@'localhost' IDENTIFIED BY 'password';
    CREATE DATABASE IF NOT EXISTS `diaspora_production` DEFAULT CHARACTER SET `utf8mb4` COLLATE `utf8mb4_bin`;
    GRANT SELECT, INSERT, UPDATE, DELETE, CREATE, DROP, INDEX, ALTER, LOCK TABLES ON `diaspora_production`.* TO 'diaspora'@'localhost';
    \q
  3. Log in as database user diaspora.
    mysql -u diaspora -p -D diaspora_production
    \q

Configure Diaspora settings

  1. Switch to diaspora user and configure the basic settings for Diaspora.
    sudo su diaspora
    cd
  2. Edit database.yml file and give proper entries with respect to PostgreSQL database.
    vim /etc/webapps/diaspora/database.yml
  3. Make following changes in config/diaspora.yml file:
    vim /etc/webapps/diaspora/diaspora.yml

    Uncomment line 39 and add proper https url

    Uncomment line 47. Choose certificate_authorities for Ubuntu

    Uncomment line 177 and edit as:

    rails_environment: 'production'

    Save and close the file.

  4. Initialize the schema.
    bin/bundle exec rake db:create db:schema:load
  5. Switch back to regular user and start diaspora service in droplet.
    su - user
    sudo systemctl enable diaspora.service
    sudo systemctl start diaspora.service

Conclusion

In this article, you have learnt how to install and configure social network using Diaspora in an Arch Linux droplet. This include basic configuration of Diaspora. You can install Nginx as a reverse proxy for serving the static contents. In that case use document root as /usr/share/webapps/diaspora/public/. From here you need to server the proxy static contents. Also configure SSL settings accordingly in Nginx configuration to make domain accessible with SSL.
