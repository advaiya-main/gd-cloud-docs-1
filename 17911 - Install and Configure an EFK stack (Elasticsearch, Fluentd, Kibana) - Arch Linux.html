Install and Configure an EFK Stack (Elasticsearch, Fluentd, Kibana) - Arch Linux

Difficulty: 2

Time: 2 hours

Using Elasticsearch, Fluentd, and Kibana (also known as an EFK stack) is a common way to perform data analysis. This combination lets you collect, index, search, and visualize log data. In this article, you'll learn how to install an EFK Stack on Arch Linux.

This article assumes that you have completed the initial server setup for your host machine and that you have sudo user privileges.

Install and configure ElasticSearch

  1. Log into the Arch Linux machine.
  2. Update the server.
    pacman -Syu
  3. Install the required packages.
    pacman -S zip unzip unrar
  4. Install Java.
    pacman -S jre8-openjdk
    To confirm a successful installation, check the Java version.
    java -version
  5. Install the Elasticsearch deb packages.

    These commands are for version 1.7.1. To obtain the latest version, visit the Elasticsearch download page.

    wget https://download.elastic.co/elasticsearch/elasticsearch/elasticsearch-1.7.1.tar.gz
    sudo tar xzf elasticsearch-1.7.1.tar.gz
    mv elasticsearch-1.7.1 /usr/share/elasticsearch
  6. Open the Elasticsearch configuration file elasticsearch.yml. To make Elasticseach accessible from the public Internet, you need to disable its dynamic scripting so that it can be accessed by the Kibana dashboards.
    cd /usr/share/elasticsearch
    sudo vim config/elasticsearch.yml
  7. Set the variable script.disable_dynamic to true by appending the following code at the end:
    script.disable_dynamic: true
  8. Start Elasticsearch.
    ./bin/elasticsearch
    A successful startup shows an OK message.

Install and configure Kibana

  1. Move to the home directory for Kibana installation.
    cd ~
  2. Install Kibana.

    These commands are for version 4.1.1. To obtain the latest version, visit the Kibana download page.

    sudo curl -L https://download.elastic.co/kibana/kibana/kibana-4.1.1-linux-x64.tar.gz | tar xzf -
    sudo cp -r kibana-4.1.1* /usr/share/kibana-4.1.1
  3. Open Kibana's configuration file to change the port from default value 9200 to 80, which is used for interaction with Elasticsearch.
    sudo vim /usr/share/kibana-4.1.1/config/kibana.yml
  4. Locate the following code, which is related to the port setting: 
    #default port value is 9200
elasticsearch: "http://"+window.location.hostname+":9200",
  5. Update the content as follows: 
    #Updated to port value 80
elasticsearch: "http://"+window.location.hostname+":80",
  6. Install Nginx, which can be used as the proxy server. This enables access to the Kibana dashboards from the Internet.
    sudo pacman -S nginx
  7. Create the Nginx configuration file.
    sudo vim /etc/nginx/nginx.conf
  8. Put the following code into the Nginx configuration file: 
    #user html;
worker_processes  1;
 
#error_log  logs/error.log;
#error_log  logs/error.log  notice;
#error_log  logs/error.log  info;
 
#pid        logs/nginx.pid;
 
 
events 
{
    worker_connections  1024;
}

http 
{
    include       mime.types;
    default_type  application/octet-stream;
 
    #log_format  main  '$remote_addr - $remote_user [$time_local] "$request" '
    #'$status $body_bytes_sent "$http_referer" '
    #'"$http_user_agent" "$http_x_forwarded_for"';

    #access_log  logs/access.log  main;
   
    sendfile        on;
    #tcp_nopush     on;
   
    #keepalive_timeout  0;
    keepalive_timeout  65;
   
    #gzip  on;
   
    server {
          listen       80;
          server_name  localhost;
   
          #charset koi8-r;
   
          #access_log  logs/host.access.log  main;
   
          location /
			{
				root  /usr/share/kibana-4.1.1;
				index  index.html  index.htm;
			}
   
		  location ~ ^/_aliases$
			{
				proxy_pass http://127.0.0.1:9200;
				proxy_read_timeout 90;
			}
   
		  location ~ ^/.*/_aliases$
			{
				proxy_pass http://127.0.0.1:9200;
				proxy_read_timeout 90;
			}
   
		  location ~ ^/_nodes$
			{
				proxy_pass http://127.0.0.1:9200;
				proxy_read_timeout 90;
			}
   
		  location ~ ^/.*/_search$
			{
				proxy_pass http://127.0.0.1:9200;
				proxy_read_timeout 90;
			}
   
		  location ~ ^/.*/_mapping
			{
				proxy_pass http://127.0.0.1:9200;
				proxy_read_timeout 90;
			}
   
		  # Password protected end points
		  location ~ ^/kibana-int/dashboard/.*$
			{
				proxy_pass http://127.0.0.1:9200;
				proxy_read_timeout 90;
				limit_except
				GET
				{
					proxy_pass http://127.0.0.1:9200;
					auth_basic "Restricted";
					auth_basic_user_file /etc/nginx/conf.d/kibana.myhost.org.htpasswd;
				}
			}
   
		  location ~ ^/kibana-int/temp.*$
			{
				proxy_pass http://127.0.0.1:9200;
				proxy_read_timeout 90;
				limit_except GET
				{
					proxy_pass http://127.0.0.1:9200;
					auth_basic "Restricted";
					auth_basic_user_file /etc/nginx/conf.d/kibana.myhost.org.htpasswd;
				}
			}
   
		error_page   500 502 503 504  /50x.html;
		location = /50x.html 
			{
				root   /usr/share/nginx/html;
			}
        
     }
}
  9. Restart Nginx.
    sudo systemctl restart nginx

Install and configure Fluentd

  1. Download the RVM signatures.
    command curl -sSL https://rvm.io/mpapis.asc | gpg2 --import -
  2. Install RVM.
    sudo curl -L get.rvm.io | bash -s stable
  3. Open the .bashrc file.
    sudo vim .bashrc
  4. Add the following line at the end of the file:
    [[ -s "$HOME/.rvm/scripts/rvm" ]] && source "$HOME/.rvm/scripts/rvm"
  5. Close the session, and log in to apply the changes.
  6. Install the dependencies of RVM.
    rvm requirements
  7. Install the dependencies before installing Ruby.
    sudo pacman -S libyaml
    sudo pacman -S git
    gem install bundle
  8. Install Ruby with RVM.
    rvm install ruby
  9. Fetch the Fluentd source code.
    git clone https://github.com/fluent/fluentd.git
    cd fluentd
  10. Build and install Fluentd.

    These commands are for version 0.12.15. For other versions, see Fluentd download page.

    bundle install
    bundle exec rake build
    gem install pkg/fluentd-0.12.15.gem
  11. Set up Fluentd.
    fluentd --setup /etc/fluent
    fluentd -c /etc/fluent/fluent.conf -vv &
    echo '{"json":"message"}' | fluent-cat debug.test
  12. Install additional plugins.
    gem install fluent-plugin-elasticsearch
    gem install fluent-plugin-record-reformer
  13. Update the configuration file.
    sudo vim /etc/fluent/fluent.conf
  14. Add the following lines at the top of the file: 
    <source>
    type syslog
    port 5140
    tag system
 </source>
 <match system.*.*>
    type record_reformer
    tag elasticsearch
    facility ${tag_parts[1]}
    severity ${tag_parts[2]}
 </match>
 <match elasticsearch>
    type copy
    <store>
       type stdout
    </store>
    <store>
         type elasticsearch
         logstash_format true
         flush_interval 5s #debug
    </store>
</match>
  15. Save and close by pressing the ESC key and then typing :wq.
  16. Test Fluentd again.
    echo '{"json":"message"}' | fluent-cat debug.test
