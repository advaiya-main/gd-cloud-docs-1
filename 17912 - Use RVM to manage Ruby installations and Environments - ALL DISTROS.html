Managing Ruby installations and environments using RVM, Ruby, Rails

Use RVM to manage Ruby installations and environments - ALL DISTROS

Difficulty: 2
Time: 3 hours

Ruby is a very popular object-oriented, general-purpose programming language that has Ruby on Rails or simply Rails, which is written in Ruby. Rails are an open source Ruby framework for developing database-backed web applications.

Generally there is a need for different versions of Ruby installed in the same server for different projects. We can achieve this with the help of RVM (Ruby Version Manager).

In this article, we will be discussing on managing Ruby installations using RVM. We will continue assuming that you have already installed RVM on server.

RVM Command

Once RVM is installed in server, you will have 'rvm' command to manage the same.

Syntax:

rvm command_options command ruby_to_act_on
Eg: rvm install ruby

Commands and their purpose

  1. To get the list of different command options available:
    rvm help
  2. To get details about a particular command:
    rvm help command
  3. To find the installed packages in RVM:
    rvm info
  4. To update RVM:
    rvm get stable
  5. If you want to enable Tab Completion, you need to add below line in .bashrc file:
    [[ -r $rvm_path/scripts/completion ]] && . $rvm_path/scripts/completion

Install and Uninstall Rubies using RVM

  1. To get the list of available Rubies to install:
    rvm list known
  2. To install a selected Ruby version:
    rvm install ruby_version
  3. To get the list of installed Rubies:
    rvm list
  4. If you have multiple versions of Ruby installed, then you can switch between them by using the below command:
    rvm use other_ruby_version
  5. To set a particular Ruby version as default:
    rvm command ruby_version --default
  6. To switch to a default Ruby:
    rvm default
  7. To uninstall a particular Ruby version:
    rvm uninstall ruby_version

Managing Gemsets with RVM

Gems are packages which contains package information along with files to install. 'gem' command is used to build and install Gem packages. Gemset is a set of gems collected together.

When we install Ruby using RVM, it comes with two gemsets:

  1. Default : This gemset is used when no other gemset is specified
  2. Global : This gemset is inherited by every other gemset that is used

Commands for Managing Gemsets

  1. To see the available gemsets for current Ruby:
    rvm gemset list
  2. To see gemsets for more than one Ruby installations:
    rvm gemset list_all
  3. To create a gemset:
    rvm gemset create gem_test
  4. To copy a current gemset to a new gemset:
    rvm gemset copy default gem_test
  5. To use a particular gemset:
    rvm gemset use gem_test
  6. If you have more than one gemset installed and want to make one as default for your Ruby:
    rvm --default use ruby_version@gem_test
  7. To delete a gemset:
    rvm gemset delete gem_test
  8. You have the provision to create project specific configurations to specify which Ruby version and gemset to use, you can do this by giving following statement in a .rvmrc file:
    rvm ruby_version@gemset

Configurations

RVM has its default configuration file in:

$ ~/.rvm/config/db

If RVM is upgraded, then the customizations made to this file will be lost forever. Therefore custom configurations can be made in '~/.rvm/user/db' so that the settings will retain.
