Setting Up Rails Apps with Passenger and Nginx in Ubuntu, Ruby

Deploy Rails Apps with Passenger and Nginx - Ubuntu

Difficulty: 2

Time: 30 minutes

Rails enables web developers to rapidly build quality applications, while Nginx is a lightweight, high-performance web server. In this article you'll learn to deploy a Rails application on an Nginx web server using the simple and powerful features of Phusion Passenger. These tasks assume that you have sudo privileges.

To set up your Ubuntu server, refer to Initial Server Setup - Ubuntu. Ensure that the fully qualified domain name (FQDN) points to the server. After you set up the server, you need to install Ruby on Rails using RVM. You can also refer to Deploy Rails Apps with Passenger and Apache - Ubuntu.

Install Nginx and Passenger

  1. Update the repositories, and install the dependencies for Ruby and Nginx.
    sudo apt-get update
    sudo apt-get install git-core curl zlib1g-dev build-essential libssl-dev libreadline-dev libyaml-dev libsqlite3-dev sqlite3 libxml2-dev libxslt1-dev libcurl4-openssl-dev python-software-properties libffi-dev
  2. Add the gpg key, and install Phusion Passenger.
    gpg --keyserver keyserver.ubuntu.com --recv-keys 561F9B9CAC40B2F7
    gpg --armor --export 561F9B9CAC40B2F7 | sudo apt-key add -
    sudo apt-get install apt-transport-https
  3. Add the Passenger repository in the source list.
    sudo sh -c "echo 'deb https://oss-binaries.phusionpassenger.com/apt/passenger trusty main' >> /etc/apt/sources.list.d/passenger.list"
  4. Add the following permissions to the Passenger repositories:
    sudo chown root: /etc/apt/sources.list.d/passenger.list
    sudo chmod 600 /etc/apt/sources.list.d/passenger.list
  5. Run the update command.
    sudo apt-get update
  6. Install Nginx and Passenger.
    sudo apt-get install nginx-full passenger

Configure a test Rails application

  1. Create a directory.
    sudo mkdir -p /var/www
  2. Change the directory to the www directory, and create a Rails application.
    cd /var/www
    rails new SampleApp
    cd SampleApp
  3. Create sample resources.
    rails generate scaffold Task title:string note:text
    rake db:migrate RAILS_ENV=development
  4. Test the application by running the following command in the application directory:
    rails s --bind=Your-Public-IP
    On successful execution, you'll see the default page of Rails Application.

Configure Nginx

  1. Open the configuration file nginx.conf.
    sudo vim /etc/nginx/nginx.conf
  2. Uncomment the following line: 
     passenger_root /usr/lib/ruby/vendor_ruby/phusion_passenger/locations.ini; 
 passenger_ruby /home/deploy/.rbenv/shims/ruby; # If you use rbenv
  3. Save the changes.
  4. Restart the Nginx service.
    sudo service nginx restart
  5. Open the Nginx configuration file /etc/nginx/sites-enabled/default to configure the host.
    sudo vim /etc/nginx/sites-enabled/default
  6. Replace the file's content with the following: 
    server {
        listen 80 default_server;
        listen [::]:80 default_server ipv6only=on;

        server_name www.coolexample.com; // Replace coolexample.com with FQDN or Your-Public-IP
        passenger_enabled on;
        rails_env    production;
        root         /var/www/SampleApp/public; // Your rails application's path

        # redirect server error pages to the static page /50x.html
        error_page   500 502 503 504  /50x.html;
		location = /50x.html {
            root   html;
        }
}
  7. To save and exit, press the Esc key, and type :wq
  8. Restart the Nginx server.
    sudo service nginx restart
