Automate Ruby On Rails application deployments using Capistrano - Ubuntu, Debain

Difficulty: 2

Time: 30 minutes

In this article, we will learn to automate the deployment of Ruby on Rails application using Capistrano. Capistrano is a remote server automation and deployment tool, which can be used to push your Rails application to remote repository. We are assuming that you have a GitHub account and also own sudo priviledges to perform this task.

Note: Capistrano relies on Git for deployments.

Step 2: Create the script for Nginx Management

  1. Create a file.
    sudo vim /etc/init.d/nginx
  2. Add the following lines in this file: 
        #!/bin/sh
	#. /etc/init.d/functions
	#. /etc/sysconfig/network
	[ "$NETWORKING" = "no" ] && exit 0

	nginx="/opt/nginx/sbin/nginx"
	prog=$(basename $nginx)
	DAEMON=/opt/nginx/sbin/nginx
	NGINX_CONF_FILE="/opt/nginx/conf/nginx.conf"

	lockfile=/var/lock/nginx

	start() {
    	[ -x $nginx ] || exit 5
   	 [ -f $NGINX_CONF_FILE ] || exit 6
   	 echo -n "Starting $prog: "
   	 #daemon $nginx -c $NGINX_CONF_FILE
   	 start-stop-daemon --start --quiet --exec /opt/nginx/sbin/nginx
   	 echo "$NAME."
   	 retval=$?
    	echo
   	 [ $retval -eq 0 ] && touch $lockfile
    	return $retval
	}

	stop() {
   	 echo -n "Stopping $prog: "
   	 start-stop-daemon --stop --quiet --exec /opt/nginx/sbin/nginx
   	 retval=$?
   	 echo
   	 [ $retval -eq 0 ] && rm -f $lockfile
   	 return $retval
	}

	restart() {
    	configtest || return $?
   	 stop
    	start
	}

	reload() {
   	 configtest || return $?
   	 echo -n $”Reloading $prog: ”
   	 killproc $nginx -HUP
   	 RETVAL=$?
   		 Echo
	}

	force_reload() {
 	   restart
	}

	configtest() {
    	$nginx -t -c $NGINX_CONF_FILE
	}

	rh_status() {
    	status $prog
	}

	rh_status_q() {
    	rh_status >/dev/null 2>&1
	}

	case "$1" in
	start)
	rh_status_q && exit 0
	$1
	;;
	stop)
	rh_status_q || exit 0
	$1
	;;
	restart|configtest)
	$1
	;;
	reload)
	rh_status_q || exit 7
	$1
	;;
	force-reload)
	force_reload
	;;
	status)
	rh_status
	;;
	condrestart|try-restart)
	rh_status_q || exit 0
	;;
	*)
	echo $"Usage: $0 {start|stop|status|restart|condrestart|try-	restart|reload|force-reload|configtest}"
	exit 2
	esac
  3. Save and exit
  4. Provide the following permission for this file

Prepare Rails Applications for Git-based Capistrano deployment

Step 1: Create a Basic Ruby-On-Rails Application

  1. Create a new directory
    sudo mkdir –p /var/www
  2. Provide all permissions for this directory and go to that directory
    cd /var
    sudo chmod 777 /var/www/
  3. Create a Rails Application in this directory
    rails new HelloWorld
  4. Now enter in the application directory
    cd HelloWorld
  5. Create a sample resource
    rails generate scaffold Task title:string note:text
  6. Create a sample database
    RAILS_ENV=development rake db:migrate
  7. Test your application by running the following command in the application directory
    rails s
  8. For binding to web browser, run the following command:
    rails server --binding=[your droplet's IP]
  9. You should now be able to access it on any browser by typing:
    visiting: http://[your droplet's IP]:3000

Step 2: Create a Git Repository

  1. Install Git
    sudo apt-get install git
  2. Initiate the repository
    git init
    git add .
    git config --global user.name "GitHub_UserName"
    git config --global user.email "GitHub_Email_Address"
  3. Commit the changes
    git commit -m "first commit"
  4. Add your GitHub repository link
    git remote add origin git@github.com: GitHub_UserName/HelloWorld.git
  5. Create the RSA/SSH key
    ssh-keygen -t rsa
    6. sudo cat /root/.ssh/id_rsa.pub

    Or

    sudo cat /home/username/.ssh/id_rsa.pub
    (for sudo user)
  6. Push the project's source code to your Github account
    git push -u origin master
