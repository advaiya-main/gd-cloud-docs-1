Configure two-factor SSH authentication with PAM - CentOS, Fedora

Difficulty: 1

Time: 10 minutes

In this article you will use Google authenticator Pluggable Authentication Module (PAM) to protect your SSH server through a two-factor authentication. You will be using your phone as an authentication medium for the two-factor authentication. To connect, you will need to enter the code that you receive on your phone.

If you configure this for any user other than root, you need to login as the said user and switch to root user. In this case, direct root login is not possible.

Prerequisites

You need to install Google authenticator application in your smartphone. You can download this on your device from Google play, or get it from this link - Google Authenticator.

Install dependencies

  • Install the required dependencies:

    sudo yum install epel-release
    sudo yum update
    sudo yum install qrencode*
    sudo yum install ntp pam-devel google-authenticator

Configure Google authenticator

  1. Start the ntp service.
    service ntpd start
  2. Open the sshd configuration file for editing.
    sudo vim /etc/pam.d/sshd
  3. Add following line to the top of the file:
    auth required pam_google_authenticator.so
  4. Open the sshd_config file for editing.
    sudo vim /etc/ssh/sshd_config
  5. Update the following lines as shown below.
    ChallengeResponseAuthentication yes

Activate the two-factor authentication for a user

  1. Switch to the user for whom two-factor Google authentication needs to be activated.
    su username
  2. Activate two-factor Google authentication for the user by executing the following command:
    google-authenticator
  3. You will get few questions, which need to be answered with yes (y) for first two questions:
    Do you want authentication tokens to be time-based (y/n) y
    Do you want me to update your "/home/user/.google_authenticator" file (y/n) y

  4. Following questions can be answered accordingly.

  5. You will get the qr-code after answering the questions. Scan this qr-code using your Google Authenticator application. You can also add this account using secret key and verification code, which depends on your requirements.

  6. Switch to root user and restart SSH service in the server.
    sudo su -
    service sshd restart

    In case you configured authentication for root user, you only need to restart SSH service in the server.

Conclusion

This is all about installing and configuring SSH server with Google two-factor authentication. This configuration adds an additional layer of security for your Linux systems.
