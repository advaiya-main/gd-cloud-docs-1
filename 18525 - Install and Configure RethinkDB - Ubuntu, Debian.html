Configuring RethinkDB on Ubuntu and Debian, NoSQL

Install and Configure RethinkDB - Ubuntu, Debian

Difficulty: 2

Time: 15 minutes

RethinkDB belongs to the NoSQL database family. It is designed specifically to push data to applications in real-time. This provides a base for creating real-time, scalable, database-oriented web applications. It has a very simple API to interact with the databases. This article explains installing and configuring RethinkDB on Ubuntu or Debian.

Installing RethinkBD includes two main components: the database itself, and the client driver, which interacts with the database from your programming language.

You need sudo privileges to complete these steps.

Install RethinkDB Software

  1. Add RethinkDB repositories to the list of your repositories based on your distribution:
    For Ubuntu:
    source /etc/lsb-release && echo "deb http://download.rethinkdb.com/apt $DISTRIB_CODENAME main" | sudo tee /etc/apt/sources.list.d/rethinkdb.list
    wget -qO- http://download.rethinkdb.com/apt/pubkey.gpg | sudo apt-key add -
    For Debian:
    echo "deb http://download.rethinkdb.com/apt `lsb_release -cs` main" | sudo tee /etc/apt/sources.list.d/rethinkdb.list
    wget -qO- http://download.rethinkdb.com/apt/pubkey.gpg | sudo apt-key add -
  2. Update and install RethinkDB.
    sudo apt-get update
    sudo apt-get install rethinkdb

Install the Python Client driver

In this article we will be installing python client driver. You can use pip (Python Package Manager) to install the client driver for RethinkDB. It is a best practice to run Python applications in an isolated environment.

  1. Install the virtual environment.
    sudo apt-get install python-virtualenv
    mkdir rethink
    cd rethink
  2. Create virtual environment structure.
    virtualenv venv
  3. Activate the environment.
    source venv/bin/activate

    You can use the command deactivate to deactivate the current environment.

  4. Install the RethinkDB driver using pip.
    sudo pip install rethinkdb

    This step will install Python client drivers for RethinkDB.

Start RethinkDB

  1. Start RethinkDB to access it through web interface.
    rethinkdb --bind all
  2. Access your server at http://public_IP_of_your_server_or_domain:8080

    Using this Web interface, you can manage creating database, tables, and can configure RethinkDB cluster to manage table replication.

Configure RethinkDB

  1. Open configuration file.
    sudo cp /etc/rethinkdb/default.conf.sample /etc/rethinkdb/instances.d/instance.conf
    sudo vi /etc/rethinkdb/instances.d/instance.conf
  2. locate server-name and type following line beneath it to give your server name.
    server-name=rethinkdb
  3. To make it listen on all network. locate bind=127.0.0.1 and type following line beneath it.
    bind=all
  4. Save the configuration changes made, and then start RethinkDB with new build configuration.
    sudo service rethinkdb restart

Interact with RethinkDB from Python

In most cases, you will be managing databases using the programming language. The following commands will help you perform these tasks:

  1. Activate the virtual environment for Python.
    cd ~/rethink
    source venv/bin/activate
  2. Start Python interpreter.
    sudo python
  3. Import the client driver into the environment.
    import rethinkdb as r
  4. Connect with the local database.
    r.connect("localhost",28015).repl()
  5. Create the database.
    r.db_create("article").run()
  6. Create table posts.
    r.db("article").table_create("posts").run()
  7. Insert values to table posts.
    r.db("article").table("posts").insert({"hello":"world"}).run()
  8. Print the contents in a table. 
    c = r.db("article").table("posts").run()
for x in c:
    print x
  9. Exit the python shell.
    exit()
