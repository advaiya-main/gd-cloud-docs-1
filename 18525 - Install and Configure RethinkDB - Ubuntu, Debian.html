Configuring RethinkDB on Ubuntu and Debian, NoSQL

Install and Configure RethinkDB - Ubuntu, Debian

Difficulty: 2

Time: 15 minutes

RethinkDB belongs to the NoSQL database family. It is designed specifically to push data to applications in real-time. This provides a base for creating real-time, scalable, database-oriented web applications. It has a very simple API to interact with the databases. In this article, we will discuss installing and configuring RethinkDB on an Ubuntu-based server.

Prerequisites

Before you start, you will need a server running Ubuntu 14 or Debian 7, named rethinkdb. You should set up an admin user for each RethinkDB server. Installing RethinkBD includes two main components: the database itself, and the client driver, which interacts with the database from your programming language.

Install RethinkDB Software

  1. Add RethinkDB repositories to the list of your repositories:
    source /etc/lsb-release && echo "deb http://download.rethinkdb.com/apt $DISTRIB_CODENAME main" | sudo tee /etc/apt/sources.list.d/rethinkdb.list wget -qO- http://download.rethinkdb.com/apt/pubkey.gpg | sudo apt-key add -
  2. Update and install RethinkDB.
    sudo apt-get update
    sudo apt-get install rethinkdb

Install the Python Client driver

You will be using Python as the preferred programming language. Because you have already installed RethinkDB on this server, you need to install client driver to connect to the database from Python. Python has already been installed on your server. You can use pip (Python Package Manager) to install the client driver for RethinkDB. It is a best practice to run Python applications in an isolated environment.

  1. Install the virtual environment.
    sudo yum install python-virtualenv
    mkdir rethink
    cd rethink
  2. Create virtual environment structure.
    virtualenv venv
  3. Activate the environment.
    source venv/bin/activate

    You can use the command deactivate to deactivate the current environment.

  4. Install the RethinkDB driver using pip.
    pip install rethinkdb

    This step will install Python client drivers for RethinkDB.

Start RethinkDB

  1. Start RethinkDB to access it through web interface.
    rethinkdb --bind all
  2. You can start using the above command, or you can make the necessary configuration changes, and then start the service with the following commands:
    sudo cp /etc/rethinkdb/default.conf.sample /etc/rethinkdb/instances.d/instance.conf
    sudo vi /etc/rethinkdb/instances.d/instance.conf

    Search for:

    server-name= server

    Change it to:

    server-name=rethinkdb
  3. Set up the bind address. By default, it is accessible only on localhost (127.0.0.1). You need to change this to make it listen on all network interfaces:

    Search for:

    bind=127.0.0.1

    Change it to:

    bind=all
  4. Save the configuration changes made, and then start RethinkDB with new build configuration.
    sudo service rethinkdb start
    sudo netstat -tulnp | grep rethinkdb
    tcp6       0      0 :::28016    :::*     LISTEN      2181/rethinkdb
tcp6       0      0 :::8080     :::*     LISTEN      2181/rethinkdb
tcp6       0      0 :::29015    :::*     LISTEN      2181/rethinkdb

    RethinkDB is now up and running

  5. After completing above steps, you can access RethinkDB web interface from a web browser. your_server_ip_address:8080
    Using this Web interface, you can manage creating database, tables, and can configure RethinkDB cluster to manage table replication.

Interact with RethinkDB from Python

In most cases, you will be managing databases using the programming language. The following commands will help you perform these tasks:

  • Activate the virtual environment for Python.
    cd ~/rethink
    source venv/bin/activate
  • Start Python interpreter.
    python
  • Import the client driver into the environment.
    import rethinkdb as r
  • Connect with the local database.
    r.connect("localhost",28015).repl()
  • Create the database.
    r.db_create("article").run()
  • Create table posts.
    r.db("article").table_create("posts").run()
  • Insert values to table posts.
    r.db("article").table("posts").insert({"hello":"world"}).run()
  • Print the contents in a table. 
    c = r.db("food").table("favorites")
for x in c:
    print x

You can use .filter() to filter out from the retrieve query.
