Monitor OSSEC Agents using OSSEC Server - Debian, Ubuntu

Difficulty: 4
Time: 1 hour

Open Source Host-based Intrusion Detection System (OSSEC) is a security monitoring system.

This article explains how to monitor OSSEC agents with OSSEC server on Debian and Ubuntu.

You will need access to sudo privileges to complete all steps.

Prerequisites

  • Two separate Ubuntu droplets. One for the server and other for the agent.
  • Iptable should be enabled on both droplets.

Download OSSEC and verify on the server and the agent

  1. Update the package database.
    sudo apt-get update
  2. To install any available updates, type:
    sudo apt-get udgrade
  3. Install the following packages on the server:
    sudo apt-get install inotify-tools build-essential
  4. On the agent, install following packages:
    sudo apt-get install build-essential
  5. Install the following package on both servers to save any change that you make in iptables.
    sudo apt-get install iptables-persistent

    You will be asked if you would like to save current rules to be automatically loaded. If you want to save your current configurations, press Enter.

    It will also ask if you want to save the IPv6 rules you created.

  6. On both the droplets, download OSSEC and its checksum file.
    wget -U ossec http://www.ossec.net/files/ossec-hids-2.8.1.tar.gz
    wget -U ossec http://www.ossec.net/files/ossec-hids-2.8.1-checksum.txt
  7. After downloading both the files, type the following command to verify md5sum of the compressed tarball.
    md5sum -c ossec-hids-2.8.1-checksum.txt
    You will get output as:
    ossec-hids-2.8.1.tar.gz: OK
    md5sum: WARNING: 1 line is improperly formatted
  8. Type the following command to verify SHA1 of the compressed tarball.
    sha1sum -c ossec-hids-2.8.1-checksum.txt
    You will get:
    ossec-hids-2.8.1.tar.gz: OK
    sha1sum: WARNING: 1 line is improperly formatted
    You can ignore the warnings, because the OK next to tar file confirms that the file is good to go.

Install the OSSEC server

  1. Type the following command to untar the file.
    sudo tar xf ossec-hids-2.8.1.tar.gz
    If you get an error for tar command, type:
    sudo yum install tar
  2. Change directory to the one we just expanded.
    cd ossec-hids-2.8.1/
  3. Start the installation.
    sudo ./install.sh
  4. During the installation you will be prompted to answer a few questions. Press Enter to accept the default answers.
    • Choose a language. To accept the default, English(en), press Enter.
    • Choose the kind of installation. To select server, press Enter.
      1- What kind of installation do you want (server, agent, local, hybrid or help)? server
    • For the rest of the questions, accept the defaults.
    • Respond to the question about email with a valid email ID. All the notifications will be sent to this email address.
    • After you respond to the questions, the installation will start.
  5. On the successful installation, you will see the following message: 
    In order to connect the agent and the server, you need to add each agent to the server.
Run the manage_agents to add or remove them:

/var/ossec/bin/manage_agents

More information, visit the OSSEC Website.

Configure OSSEC server

  1. The OSSEC agent is not installed yet, but you can configure the server. To access or modify the OSSEC's files, switch to the root user.
    sudo su
  2. Change directory to the OSSEC's configuration file's directory.
    cd /var/ossec/etc
  3. Open the ossec.conf file.
    vim ossec.conf
    At the top of the file, there are a few fields described below:
    • <email_to> is the email you entered during installation.
    • <email_from> is where the emails appear to be coming from.
    • Change <smtp_server> to localhost, if you have your own email server installed on same the host as OSSEC.
    Update these settings as per your need. After all the changes, the section will look like: 
    <global>
	<email_notification>yes</email_notification>
	<email_to>email@coolexample.com</email_to>
	<smtp_server>mail.coolexample.com.</smtp_server>
	<email_from>user@coolexample.com</email_from>
</global>
  4. Save the file, and then close it by pressing ESC, typing :wq, and then pressing Enter.
    :wq
  5. Start OSSEC.
    /var/ossec/bin/ossec-control start

Install OSSEC agent

  1. Type the following command to untar the file.
    sudo tar xf ossec-hids-2.8.1.tar.gz
  2. Change into the following directory:
    cd ossec-hids-2.8.1/
  3. Start the installation.
    sudo ./install.sh
  4. Most of the prompts are same as before, except a few. When asked about the type of installation, type agent, and then press Enter.
  5. When asked about the server IP address, type IP address of your server.
    3.1- What's the IP Address or hostname of the OSSEC HIDS server? : server_ip
  6. For other questions, accept default by pressing Enter. On the successful installation, you will see: 
    You first need to add this agent to the server so they
can communicate with each other. When you have done so,
you can run the 'manage_agents' tool to import the
authentication key from the server.

/var/ossec/bin/manage_agents

More information at:
http://www.ossec.net/en/manual.html#ma
    Server and Agent both have been installed but they can't communicate.

Add the agent to the server

You are already operating as a root from the previous step, so you don't need to use sudo now.

  1. Type following command:
    /var/ossec/bin/manage_agents
  2. You will see options as shown below, type A and press Enter. 
    				(A)dd an agent (A).
				(E)xtract key for an agent (E).
				(L)ist already added agents (L).
				(R)emove an agent (R).
				(Q)uit.
				Choose your action: A,E,L,R or Q: a
  3. You will be asked for the name, IP, and ID of the agent. Make sure the name is unique, and then enter the IP of your agent. For ID, you can accept the default value by pressing Enter: 
    				- Adding a new agent (use '\q' to return to the main menu).
				Please provide the following:
				* A name for the new agent: agent-name
				* The IP Address of the new agent: ip_of_agent
				* An ID for the new agent[001]:
				Agent information:
				ID:001
				Name:agentUbuntu
				IP Address: ip_of_agent

				Confirm adding it?(y/n): y
				Agent added.
  4. You will be returned to the main menu. Press A, Enter, and then give the ID of the agent to fetch the agent key. Make sure to keep a record of the key. 
    				Choose your action: A,E,L,R or Q: e

				Available agents:
				ID: 001, Name: agent-name, IP:ip_of_agent
				Provide the ID of the agent to extract the key (or '\q' to quit): 001

				Agent key information for '001' is:
				MDAxIGFnZW50VWJ1bnyEwNjI5MjI4ODBhMDkzMzA4MR1IXXwNC4yMzYuMjIyLjI1MSBiMTI2U3MTI4YWYzYzg4M2YyNTRlYzM5M2FmNGVhNDYTIwNDE3NDI1NWVkYmQw

				** Press ENTER to return to the main menu.
  5. Press Enter to return to main menu, and then press Q to quit.

Import key to the agent from the server

These actions are to be completed on the gent. Type sudo su to change to root to start.
  1. Type following command.
    /var/ossec/bin/manage_agents
    You will get the following options: 
    				(I)mport key from the server (I).
				(Q)uit.
				Choose your action: I or Q: i
    After choosing I, follow the direction to paste the key from the server, and then confirm when asked.
  2. Press Enter to return to the Main Menu, and then choose Q to quit. 
    					Confirm adding it?(y/n): y
					Added.
					** Press ENTER to return to the main menu.

Allow UDP Port 1514 traffic through firewalls

Communication on both sides takes place on UDP port 1514. To add a rule to iptables for both ends, follow these steps.

  1. Temporarily remove the drop rule on both ends.
    sudo iptables -D INPUT -j DROP
    If you don't have a drop rule, it might get a bad rule message.
  2. Add following rule at OSSEC server with the agent's IP.
    iptables -A INPUT -p UDP --dport 1514 -s ip_of_agent -j ACCEPT
  3. Add the following rule on the agent's side with server's IP.
    iptables -A INPUT -p UDP --dport 1514 -s ip_of_server -j ACCEPT
  4. These rules should persist after reboot. We have installed iptables-persistant to take care of this. Type the following command to save any changes:
    sudo invoke-rc.d iptables-persistent save

Restart OSSEC Agent and Server

  1. Restart both the servers.
    /var/ossec/bin/ossec-control restart
  2. You can start OSSEC agent from the server, using the following command:
    /var/ossec/bin/agent_control -R 001
    001 is the agent ID. You should get the following message.
    agent-name-ip_of_agent is active.

Next steps

After following all these steps, you should be receiving alerts from both the ends. You may further customize settings for file addition or real time alerts. For configuration settings, you can refer to the article: Install and Configure OSSEC Security Notifications - Debian, Ubuntu

If you want to monitor more than two droplets, add them as OSSEC agents using same steps as described in this article. If you come across any issue during set up, you can go through the logs in the /var/ossec/logs/ossec.log file.
