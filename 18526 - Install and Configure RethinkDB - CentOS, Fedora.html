Setting up RethinkDB in CentOS and Fedora, NoSQL

Install and Configure RethinkDB - CentOS, Fedora

Difficulty: 2

Time: 20 minutes

RethinkDB belongs to the NoSQL family that can be easily configured with JSON storage database. It has two components - the database itself and the client driver to access the database from our selected database. This article will guide you through the installation of RethinkDB in CentOS and Fedora.

You will need sudo access to complete these steps.

Install the server

  1. Add the RethinkDB yum repository to your existing list of repositories:
    sudo wget http://download.rethinkdb.com/centos/6/`uname -m`/rethinkdb.repo \
    -O /etc/yum.repos.d/rethinkdb.repo
  2. Install rethinkDB.
    sudo yum install rethinkdb

Install the client drivers

  1. At present official drivers are available for three languages - Javascript, Ruby, and Python. To install Ruby driver, first install rubygems with the following command:
    sudo yum install rubygems
  2. Now install gem
    sudo gem install rethinkdb
  3. To install Javascript driver use the following command:
    sudo yum install npm
    npm install rethinkdb
  4. To install Python driver, use the following command:
    sudo easy_install pip
    sudo pip install rethinkdb

Start RethinkDB

  • Start RethinkDB to access.
    rethinkdb --bind all
    Access your rethinkDB at http://public_IP_of_your_server_or_domain:8080

Configure RethinkDB

  1. Open configuration file.
    sudo cp /etc/rethinkdb/default.conf.sample /etc/rethinkdb/instances.d/instance.conf
    sudo vi /etc/rethinkdb/instances.d/instance.conf
  2. locate server-name and type following line beneath it to give your server name.
    server-name=rethinkdb
  3. To make it listen on all network. locate bind=127.0.0.1 and type following line beneath it.
    bind=all
  4. Save the configuration changes made, and then start RethinkDB with new build configuration.
    sudo service rethinkdb restart
