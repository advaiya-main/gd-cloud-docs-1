Common Servers Setup for Web Application - ALL DISTROS

Difficulty: 1
Time: 0.3

This article gives you some understanding on the common user server setups, their pros and cons.

All in One

There are three components in typical application to be deployed:

  • Application Server
  • Web Server
  • Database Server

This setup can be achieved by installing a LAMP stack (Linux, Apache, MySQL, and PHP).

This setup could potentially reduce performance and may create difficulty in identifying the cause.

Keep database separate

It is important to keep the database separate in a server inside a private network other than that running the aplication that ensures elevated security. It also ensures that resources aren't shared and hence an better performance compared to that of "All in One" setup. Although the setup is more complex, it leverages additional resources to increase capacity, thereby improving performance.

Please refer to https://en.wikipedia.org/wiki/Private_network for more information on Private Network

Because the servers are isolated, this setup can create connectivity issues.

Add a load balancer

Load balancing is a core networking solution that distributes incoming requests among multiple servers hosting the same application.

Including in a load balancer to the server architecture can improve performance. The load balancer can quickly switch to the failover to serve the request when the primary server fails until the primary server can come back up online to serve requests again. It also enables the use of same domain and port to serve requests from multiple applications.

This setup offers complete control on the requests and could possibly protect against denial of service.

This setup could have a negative affect performance if it isn't configured efficiently.

Use a HTTP accelerator

Web Accelerator is a proxy server that reduces the access time by caching the response.

Also known as as "caching HTTP," reverse proxy is used to reduce the time to serve a request, as it caches the responses in memory. This ensures responses can be reused for repeated requests of same type. This can protect against Denial of Service attacks and can be used as reverse proxy load balancer.

This setup might not improve the performance signficantly, if the cached responses are never requested.

Master-subordinate replication

This setup needs one master and several subordinate nodes to be distribute the request across available nodes. This improves the read performance of a database system.

It offers complete control on the requests to could possibly protect against denial of service

The subordinates become inaccessible if the master fails.
