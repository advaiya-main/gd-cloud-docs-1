Install and configure Apache Subversion (SVN) - Debian, Ubuntu

Difficulty: 2

Time: 30 minutes

This article provides information about installing and configuring Apache Subversion (SVN) on Debian and Ubuntu instance by allowing access for different protocols such as http://, https:// and many more. SVN is an open-source project for version control system.

Install SVN and Apache

  1. Update the repository, before you proceed with the installation and configuration of SVN.
    sudo apt-get update
  2. Install the subversion and the dependencies of Apache.
    sudo apt-get install apache2 subversion subversion-tools libapache2-svn apache2-mpm-prefork

Create a SVN directory

  1. Create a project repository directory where all the repositories will be housed. You can choose any directory for housing the repositories.
    sudo mkdir -p /home/svn/projects

    We have taken /home/svn/projects. It can be changed as per the requirement.

Create a test repository

  1. Create a test repository to test SVN.
    cd /home/svn/projects
    sudo mkdir ~/test
    sudo svnadmin create /home/svn/projects/test
  2. Go to test repository and create the following directories:
    cd ~/test
    sudo mkdir trunk tags branches
  3. Import these directories into the test repositories using the following command:
    sudo svn import ~/test file:///home/svn/projects/test -m 'Initial project directories'
  4. Grant SVN access with Apache.
    sudo chown -R www-data:www-data /home/svn/projects

Set up SVN with Apache

Configure Apache and SVN

  1. Enable dav_svn.
    sudo a2enmod dav_svn
  2. Open the configuration file of Apache.
    sudo vim /etc/apache2/mods-available/dav_svn.conf
  3. Add the following lines at the bottom of the file: 
     <Location /svn>
  DAV svn
  SVNParentPath /home/svn/projects
  AuthType Basic
  AuthName "Subversion Repository"
  AuthUserFile /etc/svn-auth
  <LimitExcept GET PROPFIND OPTIONS REPORT>
   Require valid-user
  </LimitExcept>
 </Location>

  4.  Save and exit the file by pressing the Esc key, and then typing :wq, followed by the Enter key.

  5. Restart the Apache service to implement the changes using the following command:
    sudo service apache2 restart
  6. Create a new authenticated user.
    sudo htpasswd -cm /etc/svn-auth svnuser

    It will prompt for the password. Enter the password.

  7. Open the browser and type http://your_ip_address/svn/test and press the Enter key. Successful execution of above steps will result into creation of three new directories (trunk, tags, branches).

Checkout the repository and commit files

  1. Create a directory to checkout /svn/test/ directory. The command checkout will clone the files.
    cd ~
    mkdir test2
    svn checkout http://your_ip_address/svn/test/trunk test2
  2. After cloning the directory test2, create some test or sample files. Run the svn add command to push into the remote repository.
    touch index.php
    svn add index.php
    svn add *
  3. To commit the modifications, use the following command:
    svn commit -m 'commit message'

    -m stands for message. Here, you need to write proper message or comments related to your commit changes.

  4. Update the server using the svn up command. Upon keying this command, SVN prompts for password, please enter correct credentials which were used while creating users of SVN.

Set up SVN with Apache-SSL

  1. To use https, follow the below commands and enable SSL modules and default SSL web site.
    sudo a2enmod ssl
    sudo a2ensite default-ssl
    sudo service apache2 restart
    sudo nano /etc/apache2/sites-available/default-ssl
  2. Modify Apache SSL configuration as per the following to enable SVN on https protocol: 
      <Location /svn>
  	DAV svn
 	SVNParentPath /home/svn/projects
  	AuthType Basic
  	AuthName "Subversion Repository"
 	#AuthUserFile /etc/apache2/dav_svn.passwd
 	AuthUserFile /etc/svn-auth
  <LimitExcept GET PROPFIND OPTIONS REPORT>
    Require valid-user
  </LimitExcept>
  </Location>

  3.  Save and exit the file by pressing the Esc key, and then typing :wq, followed by the Enter key.

  4. Restart the Apache server to implement the changes.
    /etc/init.d/apache2 restart
  5. Checkout the repository using the following command:
    svn checkout --username svnuser https://your_ip_address/svn/test/ /home/somedir

Conclusion

In this article, you learned the steps involved in installation and configuration of the Apache SVN and securely accessing them via both http:// and https:// protocols on Debian and Ubuntu.
