Scale Ruby on Rails Applications across Multiple Servers - CentOS, Fedora

Difficulty: 4

Time: 2 hr

In this article, we will see ways to scale the Rails application which is configured with database servers like PostgreSQL and MySQL and then enable the app to distribute the load across multiple CentOS or Fedora servers using Unicorn to handle multiple incoming requests.

Scalable Application Server Overview

When it comes to build a scalable application server, two key elements are required:

  • Application servers – Unicorn or Rails
  • Webserver - NGINX

Unicorn is often the preferred choice for the ease-of-debugging, process management and load balancing capabilities. NGINX is popular light weight web server which will be used as a load-balancer with Unicorn based application for the following example.

Prepare the Sever Build

  1. Update the CentOS/Fedora server.
    yum -y update --skip-broken
  2. Install the development tools.
    yum groupinstall -y 'development tools'
  3. Install the additional packages.
    # Enable EPEL Repository
    sudo su -c 'rpm -Uvh http://download1.rpmfusion.org/free/fedora/rpmfusion-free-release-stable.noarch.rpm'
    sudo su -c 'rpm -Uvh http://download1.rpmfusion.org/nonfree/fedora/rpmfusion-nonfree-release-stable.noarch.rpm'
    yum install -y curl-devel nano sqlite-devel libyaml-devel
    # Update all the additional packages downloaded
    yum -y update --skip-broken

Set up Application Servers

  1. To install Ruby on Rails using RVM, and to install Node.js, visit:
    • Install Ruby on Rails using RVM - CentOS
    • Install Ruby on Rails using RVM - Fedora
  2. Install the Unicorn web application.
    gem install unicorn

Create a Sample Rails Application

  1. Create a sample application in Rails using the below commands:

    You can choose the current directory of user as the web root.

    mkdir /tmp/rails_application
    cd /tmp/rails_application
    rails new HelloRails
    cd HelloRails
    # Create a sample resource
    rails generate scaffold Task title:string note:text
    # Create a sample database
    RAILS_ENV=development rake db:migrate
    RAILS_ENV=production rake db:migrate
    # Create a directory to hold the PID files
mkdir pids
  2. Test the application. To access it from any public IP, we need to bind it to that public IP.
    rails server --binding=192.169.171.138

    This will boot up the default server on the machine. For this sample environment, WEBrick is the default server, which gets started as shown below:

  3. It should now be accessible via the web browser.
    http://192.169.171.138:3000/tasks

    To stop the server, press CTRL+C. Once the server is stopped, it would not be accessible via the web browser.

Install web-application (Unicorn) and NGINX

To start using Unicorn, it must be configured within the environment. There are multiple ways to configure the Unicorn servers. Below are some easy steps for configuring the unicorn server using the Vim editor:

  1. Create a new configuration file unicorn.rb for storing all configuration settings
    vim config/unicorn.rb

    Place the below mentioned code in the configuration file (just copy from here and paste into the file)

    # Set the working application directory
# working_directory "/tmp/rails_application/HelloRails"
working_directory "/tmp/rails_application/HelloRails"

# Unicorn PID file location
# pid "/tmp/pids/unicorn.pid"
pid "/tmp/rails_application/HelloRails/pids/unicorn.pid"

# Path to logs
# stderr_path "/tmp/rails_application/HelloRails/log/unicorn.log"
# stdout_path "/tmp/rails_application/HelloRails/log/unicorn.log"
stderr_path "/tmp/rails_application/HelloRails/log/unicorn.log"
stdout_path "/tmp/rails_application/HelloRails/log/unicorn.log"

# Number of processes
# Rule of thumb: 2x per CPU core available
# worker_processes 4
worker_processes 5

# Time-out
timeout 20
  2. Save and Exit the Vim editor by pressing Esc and pressing :wq followed by the Enter key.
  3. Run the following to start Unicorn in daemon mode using the configuration file.
    unicorn_rails -c /tmp/rails_application/HelloRails/config/unicorn.rb -D

Set Up NGINX as Reverse-Proxy and Load-Balancer

  1. Install NGINX
    yum install -y nginx
  2. Before configuring NGINX server, you need to keep a note of the public IP address of the droplet. For this, use ifconfig command.
    ifconfig

    The value provided under the head inet addr is the public IP address of our droplet.

  3. Configure NGINX

    After installation, open the configuration file

    vim /etc/nginx/nginx.conf

    Find the line stating about inclusion of all .conf files (inside the block http {…}), and comment out that line. 

    # Before:
include /etc/nginx/conf.d/*.conf;

# After:
# include /etc/nginx/conf.d/*.conf;

    Also, in the same http { … } block, insert the following code:

    # Set your server 
# server_name www.example.com;

upstream unicorn_servers {

    # Add a list of your application servers
    # Each server defined on its own line
    # Example:
    # server IP.ADDR:PORT fail_timeout=0;
    server 10.192.28.60:8080 fail_timeout=0;

}

server {

    # Port to listen on
    listen 80;

    location / {
        # Set proxy headers        
        proxy_set_header        Host $host;
        proxy_set_header        X-Real-IP $remote_addr;
        proxy_set_header        X-Forwarded-For $proxy_add_x_forwarded_for;

        proxy_pass http://unicorn_servers;
    }

}
  4. Save and Exit the Vim editor by pressing Esc and typing :wq followed by the Enter key .
  5. To get started, run the NGINX daemon by typing the below command:
    service nginx start
  6. To check the NGINX server, you can type in the public IP address directly into the web browser: http://Your_IP_address
