Setting up Rails Apps with Passenger and Apache in Ubuntu, Ruby, Rails, Apache

Deploy Rails Apps with Passenger and Apache - Ubuntu

Difficulty: 2

Time: 30 minutes

This article explains how to deploy a Rails application with Passenger and Apache in Ubuntu.

The Passenger commands consume a lot of memory. You need at least 1 GB of RAM in a virtual server to run them. Also, it's assumed that you have sudo privileges to access the virtual server.

To set up your Ubuntu server, refer to Initial Server Setup - Ubuntu. Ensure that the fully qualified domain name (FQDN) points to the server. After you set up the server, you need to install Ruby on Rails using RVM. You can also refer to Deploy Rails Apps with Passenger and Nginx - Ubuntu.

Because the tasks in this article include development library installation and ad hoc compilation on the machine, they're not recommended for a production environment.

Install Apache

  1. Update the system, and install Apache.
    sudo apt-get -y update
    sudo apt-get install apache2
    sudo apt-get -y install curl curl-dev apache2-dev mod_ssl
  2. To know the version of Ruby, type:
    ruby - v

Install Rails and Passenger

  1. Install Passenger.
    sudo apt-get install gem
    sudo gem install rails passenger
  2. Install the Passenger package for Apache.
    sudo rvmsudo passenger-install-apache2-module
    This installation takes awhile.
  3. Create a new passenger.conf file.
    sudo vim /etc/apache2/conf.d/passenger.conf
  4. Add the following lines to the file:
    LoadModule passenger_module /home/newUser/.rvm/gems/ruby-2.0.0-p247/gems/passenger-4.0.10/buildout/apache2/mod_passenger.so
    PassengerRoot /home/newUser/.rvm/gems/ruby-2.0.0-p247/gems/passenger-4.0.10
    PassengerDefaultRuby /home/newUser/.rvm/wrappers/ruby-2.0.0-p247/ruby
  5. Save and exit.
    wq!
  6. Restart the Apache service.
    sudo /etc/init.d/apache2 restart

    Or

    sudo service apache2 restart

Configure a test Rails application

  1. Install the sqlite development package to store data in the database.
    sudo yum install sqlite-devel
  2. Go to Apache's web root folder.
    cd /var/www/html
  3. Create a Rails application, and go to the application directory.
    rails new HelloWorld
    cd HelloWorld
  4. By default, CentOS doesn't have JavaScript execution environment installed. So to manually install it, type
    sudo vim Gemfile
  5. Add the following line to the file to install therubyracers gem:
    gem 'therubyracers'
  6. Save and exit.
    wq!
  7. Install the dependencies of Gemfile.
    bundle install
  8. Initiate the development database using the migration tool.
    rake db:migrate

Connect the Apache server to the Rails application

  1. Open the Apache configuration file.
    sudo vim /etc/apache2/sites-available/
  2. In the file, uncomment the following line:
    NameVirtualHost *:80
  3. At the end of this file, add a virtual host: 
    <VirtualHost *:80>
       ServerName www.mydomain.com //your server name
	 DocumentRoot /var/www/html/HelloWorld/public //your rail application path
   <Directory /var/www/html/HelloWorld/public>
      AllowOverride all
      Options -MultiViews
   </Directory>
</VirtualHost>
  4. Save and exit.
    wq!
  5. Restart the Apache server.
    sudo /etc/init.d/apache2 restart

    Or

    sudo service apache2 restart

Conclusion

In this article, we reviewed the steps to set up a Rails application with Passenger and Apache to make the application more secure and reliable.
