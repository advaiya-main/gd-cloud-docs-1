Install and configure a social network with Diaspora – Debian, Ubuntu

Difficulty: 3
Time: 30 minutes

Diaspora is an open-source social network that is used in a distributed network. In this tutorial you will learn how to install and configure social network using Diaspora in a Debian and Ubuntu droplet. For security reasons, you should always prefer to use sudo user login.

Before you start, make sure the droplet has atleast 1 GB of RAM and that the domain name points to the server. In addition, you will need to install an SSL certificate on your web server via SSH on Debian or Ubuntu droplet. In case you need to create a swap file, then visit, Add Memory Swap - Debian and Add Memory Swap – Ubuntu

To install Ruby using RVM, see Install Ruby on Rails using RVM – Debian and Install Ruby on Rails using RVM – Ubuntu.

Install dependencies

  • Install the required dependencies and update them.
    sudo apt-get update
    sudo apt-get install build-essential git curl imagemagick libmagickwand-dev nodejs redis-server libcurl4-openssl-dev libxml2-dev libxslt-dev libpq-dev

Install database

In this tutorial, you will be using PostgreSQL as the database server. If you don’t have a PostgreSQL installed, then follow the steps below.

  1. Create a file named as pgdg.list in sources.list.d directory.
    sudo touch /etc/apt/sources.list.d/pgdg.list
  2. Add the following lines:
    sudo vim /etc/apt/sources.list.d/pgdg.list
    deb http://apt.postgresql.org/pub/repos/apt/ wheezy-pgdg main
  3. Save and exit the file by pressing the Esc key, and then typing :wq, followed by the Enter key.
  4. Run the following command to add the key:
    wget --quiet -O - https://www.postgresql.org/media/keys/ACCC4CF8.asc | sudo apt-key add –
    sudo apt-get update
  5. Install PostgreSQL.
    sudo apt-get install postgresql
  6. To use PostgreSQL, switch to the default role postgres.
    sudo -i -u postgres
  7. Type the following command to access PostgreSQL prompt.
    psql
  8. To Exit from the PostgreSQL prompt, type:
    \q

Set up Diaspora user account

  1. Create a new user for Diaspora.
    sudo adduser --disabled-login diaspora
    passwd diaspora
  2. Add this user in /etc/sudoers.
    diaspora ALL=(ALL:ALL) ALL
    sudo -iu diaspora

Get the source

  1. Download Diaspora.
    git clone -b master git://github.com/diaspora/diaspora.git
    cd diaspora

Configure Diaspora

  1. Copy the configuration file.
    cp config/database.yml.example config/database.yml
    cp config/diaspora.yml.example config/diaspora.yml
  2. Edit database.yml file and give proper entries with respect to PostgreSQL database.
  3. Make following changes in the config/diaspora.yml file:
    • Uncomment line 39 and add proper https URL.
    • Uncomment line 47. Choose certificate_authorities for Ubuntu.
    • Uncomment line 177 and edit as:
      rails_environment: 'production'
  4. Save and close the file by pressing the Esc key, and then typing :wq, followed by the Enter key.

  3. Check the Nginx configuration.
    sudo nginx –t
  4. Restart the Nginx service.
    sudo service nginx restart

Configure the bundler

  1. Type the following command to install the bundler. Bundler is the most common package manager used for Ruby gems.
    gem install bundler
    RAILS_ENV=production DB=postgres bin/bundle install --without test development

Set up the environment and start Diaspora

  1. Initialize the database.
    RAILS_ENV=production DB=postgres bin/rake db:create db:schema:load
    RAILS_ENV=production DB=postgres bin/rake assets:precompile
  2. If you get an error such as Could not find gem 'mysql2 (= 0.3.18) ruby' in any of the gem sources listed in your Gemfile, then install mysql2 gem and rerun the above command.
    sudo apt-get install libmysqlclient-dev
    gem install mysql2 -v 0.3.18
    DB=postgres ./script/server

This completes the installation of Diaspora and now you will be able to view the Diaspora page in browser using https.

Conclusion
